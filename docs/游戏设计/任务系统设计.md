# 任务系统设计文档

## 一、系统概述

### 1.1 系统定位
任务系统是《苍穹劫·摸金传人》的核心引导与奖励驱动系统，通过日常任务、周常任务和成就系统三位一体的设计，激励玩家持续参与游戏内容，提升用户粘性与付费转化。

### 1.2 设计目标
- **活跃驱动**：通过日常/周常任务确保每日登录
- **目标引导**：通过成就系统提供长期追求目标
- **奖励激励**：通过多层次奖励提升玩家成就感
- **社交促进**：通过协作任务增强玩家互动

## 二、核心玩法设计

### 2.1 任务类型分类

#### 战斗类任务
| 任务类型 | 描述 | 代表任务 |
|---------|------|---------|
| 通关任务 | 完成特定关卡 | 通关第10章/精英关卡1次 |
| 击杀任务 | 击败敌人数量 | 击败100个精英怪物 |
| BOSS任务 | 击败BOSS | 击败3次世界BOSS |
| 竞技任务 | 参与PVP | 在竞技场完成5场战斗 |

#### 养成类任务
| 任务类型 | 描述 | 代表任务 |
|---------|------|---------|
| 升级任务 | 角色/装备升级 | 将任意角色升至50级 |
| 进阶任务 | 角色/装备进阶 | 完成角色3次进阶 |
| 技能任务 | 技能升级 | 升级技能至满级 |
| 装备任务 | 装备强化/精炼 | 强化装备至+10 |

#### 社交类任务
| 任务类型 | 描述 | 代表任务 |
|---------|------|---------|
| 好友任务 | 社交互动 | 添加3个好友 |
| 公会任务 | 公会活动参与 | 参与公会副本1次 |
| 组队任务 | 多人副本 | 邀请好友组队通关 |
| 协助任务 | 帮助其他玩家 | 协助好友通关1次 |

#### 资源类任务
| 任务类型 | 描述 | 代表任务 |
|---------|------|---------|
| 收集任务 | 收集特定物品 | 收集10件紫色装备 |
| 消费任务 | 消耗游戏资源 | 消耗10000金币 |
| 交易任务 | 市场交易 | 在市场上架物品 |
| 生产任务 | 制作装备/道具 | 制作1件橙色装备 |

### 3.3 任务进度追踪

#### 进度计算逻辑
```
进度更新触发点：
1. 玩家行为事件触发（如：完成关卡、击杀怪物）
2. 后台定时检测（如：每日活跃度计算）
3. 手动刷新（如：玩家点击刷新按钮）

进度同步机制：
- 实时更新：战斗类任务立即刷新
- 批量更新：养成类任务按批次处理
- 异步更新：社交类任务延迟同步
```

#### 进度显示规则
| 进度状态 | 显示样式 | 交互行为 |
|---------|---------|---------|
| 未开始 | 灰色进度条 | 显示任务详情 |
| 进行中 | 绿色进度条 | 显示当前/目标 |
| 已完成(未领取) | 金色闪烁 | 点击领取奖励 |
| 已完成(已领取) | 灰色勾选 | 仅显示已完成 |

### 3.4 任务奖励领取

#### 领取机制
- **自动领取**：玩家登录时自动发放日常任务奖励
- **手动领取**：玩家需点击领取按钮
- **一键领取**：VIP用户可一键领取所有已完成任务

#### 奖励发放规则
```
奖励发放优先级：
1. 物品先进入背包，超出上限则发送邮件
2. 货币直接增加到玩家账户
3. 稀有道具增加获得记录
4. 称号自动装备或放入称号栏
```

## 五、数据表设计

### 5.1 日常任务表 (daily_tasks)

```sql
CREATE TABLE `daily_tasks` (
  `task_id` VARCHAR(32) PRIMARY KEY COMMENT '任务ID',
  `task_name` VARCHAR(64) NOT NULL COMMENT '任务名称',
  `task_desc` VARCHAR(256) COMMENT '任务描述',
  `task_type` TINYINT NOT NULL COMMENT '任务类型：1战斗/2养成/3社交/4资源',
  `task_subtype` VARCHAR(32) COMMENT '任务子类型',
  `task_target` INT NOT NULL COMMENT '目标数量',
  `task_target_id` VARCHAR(64) COMMENT '目标关联ID（如关卡ID）',
  `task_target_param` VARCHAR(256) COMMENT '目标参数（JSON格式）',
  `difficulty` TINYINT DEFAULT 1 COMMENT '难度等级1-5',
  `reward_gold` INT DEFAULT 0 COMMENT '金币奖励',
  `reward_exp` INT DEFAULT 0 COMMENT '经验奖励',
  `reward_diamond` INT DEFAULT 0 COMMENT '钻石奖励',
  `reward_item_id` VARCHAR(32) COMMENT '道具奖励ID',
  `reward_item_count` INT DEFAULT 0 COMMENT '道具奖励数量',
  `reward_type` VARCHAR(32) COMMENT '奖励类型标签',
  `priority` INT DEFAULT 100 COMMENT '显示优先级',
  `unlock_level` INT DEFAULT 1 COMMENT '解锁等级',
  `unlock_quest` VARCHAR(32) COMMENT '前置任务ID',
  `is_repeatable` TINYINT DEFAULT 0 COMMENT '是否可重复完成',
  `reset_type` VARCHAR(16) DEFAULT 'daily' COMMENT '重置类型',
  `is_active` TINYINT DEFAULT 1 COMMENT '是否启用',
  `start_time` DATETIME COMMENT '开始时间',
  `end_time` DATETIME COMMENT '结束时间',
  `sort_order` INT DEFAULT 0 COMMENT '排序',
  `version` INT DEFAULT 1 COMMENT '版本号',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX `idx_type` (`task_type`),
  INDEX `idx_reset_type` (`reset_type`),
  INDEX `idx_active` (`is_active`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='日常任务配置表';
```

### 6.2 具体关联设计

#### 与战斗系统关联
- 任务进度触发战斗事件（如通关关卡）
- 战斗结算时调用任务进度更新接口
- 战斗获得的数据（伤害、通关时间）用于成就计算

#### 与养成系统关联
- 养成操作触发养成类任务进度
- 角色升级、装备强化等事件上报
- 养成资源消耗计入消费类任务

#### 与社交系统关联
- 好友互动触发社交类任务
- 公会活动完成触发公会任务
- 组队信息用于协助类任务判定

#### 与邮件系统关联
- 背包满时奖励自动发送邮件
- 批量奖励通过邮件系统发放
- 邮件附件包含任务奖励领取
