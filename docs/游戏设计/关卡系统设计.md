## 一、地图生成规则

### 1.1 随机地图生成

#### 生成算法流程
```
1. 初始化空地图（全地板）
2. 设置边界墙壁
3. 放置玩家起点 (1, 1)
4. 随机放置事件：
   - 怪物：根据关卡配置的概率分布
   - 宝箱：根据关卡配置的数量
   - 机关：三种类型按比例分配
5. 验证路径连通性（BFS检查）
6. 放置出口（距离起点最远的有效格子）
```

#### 随机地图配置

| 关卡 | 地图大小 | 怪物概率 | 宝箱概率 | 机关概率 | 墙壁密度 |
|------|----------|----------|----------|----------|----------|
| 第1关 | 20×20 | 5% | 4% | 4%/4%/4% | 15% |
| 第2关 | 16×16 | 5% | 5% | 4%/4%/4% | 20% |
| 第3关 | 固定 | - | - | - | - |

### 1.2 固定地图生成

#### 触发条件
- 当前关卡配置 `type: 'fixed'`
- 或特定Boss关卡（如第3关）

#### 加载流程
```
1. 读取预设地图数据（JSON格式）
2. 解析格子类型
3. 初始化事件监听
4. 放置玩家起点
5. 预计算路径缓存
```

#### 固定地图数据结构
```javascript
const FIXED_MAP = {
    size: 16,
    data: [
        // 行数据，每行16个格子
        "WWWWWWWWWWWWWWWW",
        "W·············W",
        "W·E·········E·W",
        // ... 继续定义
    ],
    playerStart: { x: 1, y: 1 },
    boss: { x: 8, y: 11 },
    exit: { x: 15, y: 14 }
};
```

### 1.3 地图格子类型

| 类型标识 | 名称 | 是否可通行 | 说明 |
|----------|------|------------|------|
| `floor` | 地板 | ✅ | 普通可通行区域 |
| `wall` | 墙壁 | ❌ | 不可通行 |
| `player` | 玩家 | ✅ | 玩家当前位置 |
| `enemy` | 敌人 | ⚔️ | 触发战斗 |
| `boss` | Boss | ⚔️ | 触发Boss战 |
| `chest` | 宝箱 | ✅ | 拾取金币 |
| `stairs` | 楼梯 | ✅ | 通关出口 |
| `mechanism-*` | 机关 | ✅ | 触发机关互动 |
| `buff-*` | Buff | ✅ | 触发Buff效果 |
| `story` | 故事碎片 | ✅ | 触发剧情 |
| `coffin` | 星棺 | ⚔️ | 触发Boss战 |

## 二、自动寻路系统

### 2.1 寻路算法选择

采用 **A*算法** 作为核心寻路算法，原因：
- 效率高：综合考虑距离和预估代价
- 路径最优：在启发式函数设计合理时能找到最短路径
- 可扩展：易于添加额外约束条件

### 2.2 A*算法实现

#### 核心数据结构
```javascript
// 节点定义
class Node {
    x, y           // 坐标
    gCost          // 从起点到当前节点的实际代价
    hCost          // 从当前节点到终点的预估代价（启发式）
    fCost          // gCost + hCost
    parent         // 父节点（用于路径回溯）
}

// 启发式函数（曼哈顿距离）
function heuristic(a, b) {
    return Math.abs(a.x - b.x) + Math.abs(a.y - b.y);
}
```

#### 寻路流程
```
1. 初始化开放列表和关闭列表
2. 将起点加入开放列表
3. 循环直到：
   a. 开放列表为空 → 无路径可通
   b. 找到终点 → 回溯构建路径
4. 从开放列表选择fCost最小的节点
5. 检查相邻格子（上下左右）
6. 更新节点的gCost和fCost
7. 重复步骤4-6
```

### 2.3 避障规则

#### 必须避开的格子类型
| 格子类型 | 行为 |
|----------|------|
| `wall` | 完全不可通行 |
| `enemy` | 主动绕开，不触发战斗 |
| `boss` | 主动绕开，不触发Boss战 |
| `coffin` | 主动绕开，不触发Boss战 |

#### 可穿越的格子类型
| 格子类型 | 行为 |
|----------|------|
| `floor` | 正常通行 |
| `chest` | 通行并触发拾取 |
| `mechanism-*` | 通行并触发机关 |
| `buff-*` | 通行并获得Buff |
| `story` | 通行并触发剧情 |
| `stairs` | 通行并触发结算 |

### 2.4 路径可视化

- 计算出路径后，高亮显示路径格子
- 使用不同颜色区分：
  - 普通路径：浅蓝色
  - 危险路径（经过机关）：橙色
  - 最短路径：绿色

### 2.5 移动动画

```
移动间隔：300ms/格
动画类型：平滑滑动
取消条件：点击取消 / 遭遇战斗 / 疲劳耗尽
```

## 三、移动规则

### 3.1 相邻格移动

| 操作 | 触发方式 | 移动范围 |
|------|----------|----------|
| 点击移动 | 点击相邻格子 | 1格 |
| 自动寻路 | 点击远处格子 | 多格 |

### 3.2 视野揭示机制

#### 基础规则
- 视野半径：2格（以玩家为中心）
- 视野形状：菱形/方形

#### 视野揭示公式
```
对于坐标 (x, y) 的格子，如果满足：
|dx| + |dy| <= 视野半径
则揭示该格子
```

#### 视野效果
- **已揭示**：显示实际内容
- **未揭示**：显示迷雾 `?`

### 3.3 移动触发事件

| 格子类型 | 触发结果 |
|----------|----------|
| `floor` | 正常移动 |
| `enemy` | 触发战斗 |
| `boss` | 触发Boss战 |
| `chest` | 获得金币 |
| `mechanism-*` | 打开机关界面 |
| `buff-*` | 获得Buff效果 |
| `stairs` | 通关结算 |

## 四、事件触发规则

### 4.1 事件触发流程

```
1. 玩家移动到目标格子
2. 判定格子类型
3. 执行对应事件逻辑
4. 更新游戏状态
5. 刷新UI显示
```

### 4.2 事件类型定义

#### 战斗事件
```javascript
const BATTLE_EVENTS = ['enemy', 'boss', 'coffin'];

// 触发条件：踩到对应格子
// 处理流程：
// 1. 暂停移动
// 2. 初始化战斗界面
// 3. 执行战斗逻辑
// 4. 战斗结束后继续/结算
```

#### 机关事件
```javascript
const MECHANISM_EVENTS = ['mechanism-luck', 'mechanism-knowledge', 'mechanism-hero'];

// 触发条件：踩到机关格子
// 处理流程：
// 1. 弹出机关互动界面
// 2. 玩家选择选项
// 3. 根据选项执行结果
// 4. 机关格子变为地板
```

#### Buff事件
```javascript
const BUFF_EVENTS = ['buff-attack', 'buff-fatigue'];

// 触发条件：踩到Buff格子
// 效果：
// - buff-attack：攻击力+20%，持续当前关卡
// - buff-fatigue：疲劳度+30点
```

### 4.3 事件冷却机制

- 机关事件：每个机关只能触发一次
- Buff事件：某些Buff可叠加，某些不可叠加
- 战斗事件：击败后可重新进入该格子

## 五、战斗触发条件

### 5.1 战斗触发场景

| 场景 | 触发条件 | 战斗类型 |
|------|----------|----------|
| 遭遇普通敌人 | 移动到enemy格子 | 普通战斗 |
| 遭遇Boss | 移动到boss/coffin格子 | Boss战 |
| 追击系统 | 敌人相邻时自动追击 | 强制战斗 |

### 5.2 战斗初始化

```javascript
function startBattle(isBoss = false) {
    // 1. 暂停地图移动
    gameState.isInBattle = true;
    
    // 2. 初始化战斗单位
    initBattleHeroes();      // 初始化英雄
    initBattleEnemies(isBoss); // 初始化敌人
    
    // 3. 构建行动队列（按速度排序）
    buildActionQueue();
    
    // 4. 显示战斗界面
    showBattleModal();
    
    // 5. 开始自动战斗
    autoBattleLoop();
}
```

### 5.3 战斗结束处理

#### 胜利条件
- 所有敌人血量归零

#### 失败条件
- 所有英雄血量归零

#### 胜利后处理
```javascript
function onBattleVictory() {
    // 1. 统计奖励
    const goldReward = isBoss ? 100 : 50;
    
    // 2. 计算疲劳消耗
    const fatigueCost = calculateFatigueCost();
    
    // 3. 恢复英雄HP
    restoreHeroHP();
    
    // 4. 更新关卡进度
    updateLevelProgress();
    
    // 5. 关闭战斗界面
    hideBattleModal();
    
    // 6. 继续探索/结算
    continueExploration();
}
```

### 6.1 疲劳度定义

| 属性 | 说明 |
|------|------|
| 初始值 | 100 |
| 最大值 | 100 |
| 最低值 | 0（无法继续探索） |

| 事件 | 消耗值 |

## 七、Buff系统

### 7.1 Buff类型定义

| BuffID | 名称 | 图标 | 效果 | 持续时间 |
|--------|------|------|------|----------|
| `buff-attack` | 攻击强化 | ⚔️ | 攻击力+20% | 当前关卡 |
| `buff-fatigue` | 疲劳恢复 | 💚 | 疲劳+30 | 即时 |
| `buff-defense` | 防御强化 | 🛡️ | 防御+15% | 当前关卡 |
| `buff-speed` | 加速移动 | ⚡ | 移动速度+20% | 当前关卡 |
| `buff-vision` | 视野扩大 | 👁️ | 视野半径+1 | 当前关卡 |

### 7.2 Buff效果叠加规则

| Buff类型 | 叠加规则 |
|----------|----------|
| 攻击强化 | 不同类型可叠加，同类型取最大值 |
| 防御强化 | 不同类型可叠加，同类型取最大值 |
| 疲劳恢复 | 不可叠加，取最新值 |
| 移动加速 | 可叠加（乘法计算） |
| 视野扩大 | 不可叠加，取最大值 |

### 7.3 Buff触发逻辑

```javascript
function triggerBuff(buffId, position) {
    // 1. 检查Buff是否已存在
    const existingBuff = gameState.activeBuffs.find(b => b.id === buffId);
    
    // 2. 根据叠加规则处理
    switch(buffId) {
        case 'buff-attack':
        case 'buff-defense':
            if (existingBuff) {
                existingBuff.value = Math.max(existingBuff.value, newValue);
            } else {
                gameState.activeBuffs.push({ id: buffId, value: newValue });
            }
            break;
        case 'buff-fatigue':
            gameState.fatigue = Math.min(gameState.maxFatigue, 
                                        gameState.fatigue + 30);
            break;
        // ...
    }
    
    // 3. 消耗Buff格子（变为地板）
    gameState.map[position.y][position.x].type = 'floor';
    
    // 4. 更新UI
    updateBuffDisplay();
}
```
