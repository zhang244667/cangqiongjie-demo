# 剧情演出系统设计

> 《苍穹劫》视觉小说式剧情表现系统

---

## 一、系统定位

剧情演出系统用于在关键时刻播放剧情动画，增强游戏的叙事体验。

### 1.1 触发时机

| 场景 | 触发条件 | 内容 |
|------|----------|------|
| 关卡开场 | 进入关卡时 | 介绍关卡背景、氛围渲染 |
| Boss战前 | 踩到Boss格子 | Boss出场演出、对话 |
| 战斗胜利 | 击败Boss | 胜利结算、获得奖励 |
| 战斗失败 | 全灭 | 失败结算、鼓励重试 |

### 1.2 设计目标

- 提供沉浸式的叙事体验
- 增强Boss战的仪式感
- 丰富游戏的世界观呈现
- 不阻塞核心玩法，可快速跳过

### 2.2 自适应策略

| 屏幕方向 | 背景图处理 | 立绘位置 | 对话框 |
|----------|------------|----------|--------|
| 竖屏 | cover填充，居中裁切 | 中下区域 | 底部，高度30% |
| 横屏 | cover填充，居中裁切 | 左右两侧 | 底部，高度25% |

### 2.2 画面分层

| 层级 | 内容 | z-index |
|------|------|---------|
| 背景层 | 渐变背景 + 场景图片 | 1 |
| 特效层 | 震动、闪烁、粒子 | 5 |
| 立绘层 | 角色立绘图片 | 10 |
| 对话框 | 角色名 + 对话文字 | 20 |
| 选择层 | 分支选择按钮 | 30 |
| 提示层 | "点击继续"提示 | 25 |

## 三、交互流程

### 3.1 完整流程

```
玩家进入关卡
    ↓
背景渐入（0.8秒）
    ↓
对话框出现
    ↓
打字机效果逐字显示（30ms/字）
    ↓
【等待玩家点击】
    ├─ 点击时正在打字 → 跳过打字，显示完整文本
    └─ 点击时已打完 → 进入下一条对话
    ↓
对话结束
    ├─ 有选择分支 → 显示选择按钮
    │   ├─ 点击"准备战斗" → 开始战斗
    │   └─ 点击其他选项 → 继续剧情
    └─ 无选择分支 → 自动关闭/触发回调
    ↓
场景渐出（0.5秒）
    ↓
进入游戏/战斗
```

### 3.2 交互规范

| 操作 | 效果 |
|------|------|
| 点击任意位置（打字中） | 跳过打字，显示完整文本 |
| 点击任意位置（打字完成） | 进入下一条对话 |
| 点击选择按钮 | 执行对应动作 |
| 快速连续点击 | 依次跳过，不会卡死 |

## 四、配置规范

### 4.1 剧情配置结构

```javascript
{
    background: 'tomb_entrance',  // 场景背景ID
    music: 'tense',               // 背景音乐（可选）
    effects: ['shake', 'flash'],  // 特效列表（可选）
    dialogues: [                  // 对话列表
        {
            speaker: '旁白',       // 说话者名称
            text: '...',          // 对话内容
            position: 'center',   // 立绘位置：left/center/right
            isBoss: false         // 是否Boss（红色名字）
        }
    ],
    choices: [                    // 选择分支（可选）
        { text: '准备战斗！', action: 'startBattle' }
    ]
}
```

## 六、场景背景图

### 6.1 场景列表

| 场景ID | 文件名 | 用途 | 核心元素 |
|--------|--------|------|----------|
| tomb_entrance | tomb_entrance.jpg | 关卡入口 | 破旧石门、微弱光线、青苔 |
| ancient_corridor | ancient_corridor.jpg | 走廊场景 | 狭长走廊、火把、青苔石壁 |
| starlight_hall | starlight_hall.jpg | 大厅场景 | 星象图穹顶、发光星盘 |
| boss_room | boss_room.jpg | Boss房 | 悬浮星棺、宇宙能量 |
| victory_scene | victory_scene.jpg | 胜利结算 | 宝藏、金色光芒 |

### 6.2 图片规范

**目标规格**（移动端优先）：
- 比例：9:16竖版
- 推荐：1080×1920 或 1440×2560
- 风格：写实国漫、暗黑奇幻
- 要求：无角色、纯背景

**现有图片**（临时横版）：
- 当前：16:9横版（2848×1600）
- 处理：用`background-size:cover`自适应填充
- 后续：逐步替换为9:16竖版

## 七、技术实现

### 7.1 核心类

- `StoryScene` - 单个场景管理
- `StoryPlayer` - 整体流程管理

### 7.2 主要函数

```javascript
// 播放剧情
playStory(scenes, onComplete)

// 关卡开场
playLevelIntro(levelId, onComplete)

// Boss出场
playBossIntro(levelId, onComplete)

// 胜利结算
playVictoryStory(onComplete)

// 失败结算
playDefeatStory(onComplete)
```

### 7.3 模块化

剧情演出系统已模块化，可独立引用：

```html
<script src="剧情演出系统.js"></script>
```
