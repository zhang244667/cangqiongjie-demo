# 邮件系统设计文档

## 一、系统概述

### 1.1 系统定位
邮件系统是《苍穹劫·摸金传人》的核心信息传递与奖励发放系统，承担着系统通知、奖励发放、玩家间私信等功能。系统设计兼顾运营效率与用户体验，支持自动化邮件发送与手动邮件管理。

### 1.2 设计目标
- **信息传达**：及时传递系统公告、活动通知
- **奖励发放**：可靠送达游戏奖励
- **社交互动**：支持玩家间私信交流
- **数据追踪**：完整记录邮件生命周期

## 二、核心玩法设计

### 2.1 邮件类型详细设计

#### 2.1.1 系统邮件类型

| 邮件类型 | 触发方式 | 发送范围 | 附件支持 | 有效期 |
|---------|---------|---------|---------|-------|
| 全服公告 | 运营手动发送 | 所有玩家 | 否 | 可配置 |
| 等级奖励 | 自动触发 | 单个玩家 | 是 | 30天 |
| 活动奖励 | 自动触发 | 符合条件的玩家 | 是 | 7天 |
| 充值凭证 | 自动触发 | 单个玩家 | 是 | 永久 |
| 补偿邮件 | 手动发送 | 指定玩家 | 是 | 可配置 |
| 订单邮件 | 自动触发 | 单个玩家 | 是 | 永久 |

#### 2.1.2 个人邮件类型

| 邮件类型 | 发送限制 | 附件支持 | 审核机制 | 有效期 |
|---------|---------|---------|---------|-------|
| 玩家私信 | 需好友关系 | 否 | 内容审核 | 7天 |
| 客服回复 | 无限制 | 是 | 客服审核 | 30天 |
| 举报反馈 | 无限制 | 是 | 自动处理 | 30天 |
| 公会邮件 | 会长/副会长 | 是 | 无 | 14天 |

### 2.2 邮件奖励机制

#### 2.2.1 附件奖励类型
```
附件奖励支持类型：
├── 货币类
│   ├── 金币
│   ├── 钻石
│   ├── 绑定钻石
│   └── 月卡货币
│
├── 道具类
│   ├── 消耗品（药水、材料）
│   ├── 装备（白/绿/蓝/紫/橙/红）
│   ├── 召唤券（普通/精致/稀有/史诗/传说）
│   └── 礼包（各级别礼包盒）
│
├── 功能类
│   ├── 经验道具
│   ├── 体力道具
│   ├── 补签卡
│   └── 刷新卡
│
└── 特殊类
    ├── 称号
    ├── 头像框
    ├── 皮肤
    └── 特效
```

#### 2.2.2 奖励发放规则
```
发放规则：
├── 背包空间检测
│   ├── 空间足够 → 直接发放
│   ├── 空间不足 → 部分发放+邮件补发
│   └── 超量道具 → 全额邮件发放
│
├── 附件领取
│   ├── 点击领取 → 逐个领取
│   ├── 一键领取 → 批量领取
│   └── 批量上限 → 每次最多50个附件
│
└── 邮件状态
    ├── 未读 → 有附件可领
    ├── 已读 → 无附件
    └── 已领取 → 附件已领取
```

### 2.3 邮件过期机制

```
过期规则：
├── 有效期计算
│   ├── 从发送时间开始计算
│   ├── 系统邮件：7-30天
│   ├── 个人邮件：7天
│   └── 永久邮件：无过期
│
├── 过期前提醒
│   ├── 过期前3天：首次提醒
│   ├── 过期前1天：再次提醒
│   └── 过期当天：最后提醒
│
└── 过期处理
    ├── 未领取附件自动作废
    ├── 邮件内容保留展示
    └── 过期邮件标记显示
```

## 三、详细功能设计

```yaml
mail_template:
  模板配置:
    template_id: "MT001"
    template_name: "新手礼包"
    mail_type: "reward"              # system/reward/announcement/private
    priority: 1                     # 显示优先级
    title_template: "🎁 欢迎来到苍穹！"  # 标题模板
    content_template: "欢迎入驻苍穹大陆，这是您的新手礼包，请查收。"  # 内容模板
    
  附件配置:
    attachments:
      - item_id: "GOLD"
        amount: 1000
      - item_id: "NEWBIE_BOX"
        amount: 1
        
  有效期配置:
    validity_days: 7                # 有效期(天)
    expire_notice: true             # 是否过期提醒
    
  发送配置:
    send_type: "auto"              # auto/manual
    trigger_condition: null         # 触发条件(自动发送时)
```

```

└──────┬──────┘
       │

┌─────────────┐
│  参数校验   │  ← 接收者ID、附件、类型
└──────┬──────┘

       ▼

       │
       ▼
┌─────────────┐
│  数据存储   │  ← 写入邮件表、附件表
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  推送通知   │  ← 红点提示、推送通知
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  完成       │
└─────────────┘
```

#### 3.2.2 发送限制
```yaml
发送限制规则:
  单个玩家:
    - 每日邮件上限: 100封
    - 每小时邮件上限: 20封
    - 附件总格子: 50个/封
    
  全服邮件:
    - 发送间隔: 最少5分钟
    - 单次接收者: 无上限
    - 需要审核: 是
    
  个人邮件:
    - 需要好友: 是(私信)
    - 内容审核: 是
    - 附件限制: 否
```

### 3.3 邮件领取机制

#### 3.3.1 领取规则
```yaml
领取规则:
  基本规则:
    - 附件一键领取: VIP3解锁
    - 单封领取: 无限制
    - 领取验证: 在线状态检测
    
  附件规则:
    - 部分领取: 不支持
    - 超量处理: 发送新邮件
    - 重复领取: 禁止
    
  批量领取:
    - 最多批量: 50封
    - 批量条件: 已读+有附件
    - VIP加成: VIP5可批量100封
```

```
       ▼

│  验证背包   │  ← 背包空间是否足够
└──────┬──────┘
       │
       ├──────────────────┐
       │                  │
    空间足够           空间不足
       │                  │
       ▼                  ▼
┌─────────────┐     ┌─────────────┐
│  发放奖励   │     │ 部分发放+   │
│  更新状态   │     │ 新邮件补发  │
└──────┬──────┘     └──────┬──────┘
       │                  │
       ▼                  ▼
┌─────────────┐     ┌─────────────┐
```

## 五、数据表设计

### 5.1 邮件模板表 (mail_templates)

```sql
CREATE TABLE `mail_templates` (
  `template_id` VARCHAR(32) PRIMARY KEY COMMENT '模板ID',
  `template_name` VARCHAR(64) NOT NULL COMMENT '模板名称',
  `mail_type` VARCHAR(16) NOT NULL COMMENT '邮件类型',
  `mail_subtype` VARCHAR(32) COMMENT '邮件子类型',
  `priority` INT DEFAULT 100 COMMENT '显示优先级',
  `title_template` VARCHAR(128) NOT NULL COMMENT '标题模板',
  `content_template` TEXT COMMENT '内容模板',
  `attachment_config` JSON COMMENT '附件配置',
  `validity_days` INT DEFAULT 7 COMMENT '有效期(天)',
  `send_type` VARCHAR(16) DEFAULT 'manual' COMMENT '发送类型：auto/manual',
  `trigger_event` VARCHAR(64) COMMENT '触发事件',
  `trigger_params` JSON COMMENT '触发参数',
  `is_active` TINYINT DEFAULT 1 COMMENT '是否启用',
  `is_global` TINYINT DEFAULT 0 COMMENT '是否全服模板',
  `created_by` BIGINT COMMENT '创建人',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX `idx_type` (`mail_type`),
  INDEX `idx_active` (`is_active`),
  INDEX `idx_global` (`is_global`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='邮件模板表';
```

### 5.2 用户邮件表 (user_mails)

```sql
CREATE TABLE `user_mails` (
  `mail_id` BIGINT AUTO_INCREMENT PRIMARY KEY,
  `mail_uuid` VARCHAR(64) NOT NULL COMMENT '邮件唯一ID',
  `user_id` BIGINT NOT NULL COMMENT '接收者ID',
  `sender_id` BIGINT DEFAULT 0 COMMENT '发送者ID(0为系统)',
  `sender_name` VARCHAR(32) COMMENT '发送者名称',
  `sender_type` VARCHAR(16) DEFAULT 'system' COMMENT '发送者类型：system/player/gm',
  `mail_type` VARCHAR(16) NOT NULL COMMENT '邮件类型',
  `mail_subtype` VARCHAR(32) COMMENT '邮件子类型',
  `title` VARCHAR(128) NOT NULL COMMENT '邮件标题',
  `content` TEXT COMMENT '邮件内容',
  `attachment_count` INT DEFAULT 0 COMMENT '附件数量',
  `has_attachment` TINYINT DEFAULT 0 COMMENT '是否有附件',
  `attachment_claimed` TINYINT DEFAULT 0 COMMENT '附件是否已领取',
  `attachment_claimed_time` DATETIME COMMENT '附件领取时间',
  `is_read` TINYINT DEFAULT 0 COMMENT '是否已读',
  `read_time` DATETIME COMMENT '阅读时间',
  `priority` INT DEFAULT 100 COMMENT '优先级',
  `validity_days` INT DEFAULT 7 COMMENT '有效期(天)',
  `expire_time` DATETIME COMMENT '过期时间',
  `is_expired` TINYINT DEFAULT 0 COMMENT '是否已过期',
  `is_deleted` TINYINT DEFAULT 0 COMMENT '是否已删除',
  `delete_time` DATETIME COMMENT '删除时间',
  `status` VARCHAR(16) DEFAULT 'pending' COMMENT '状态：pending/sent/read/claimed/expired',
  `source` VARCHAR(32) COMMENT '来源(活动/订单/GM等)',
  `source_id` VARCHAR(64) COMMENT '来源ID',
  `extend_data` JSON COMMENT '扩展数据',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY `uk_mail_uuid` (`mail_uuid`),
  INDEX `idx_user` (`user_id`),
  INDEX `idx_user_type` (`user_id`, `mail_type`),
  INDEX `idx_user_unread` (`user_id`, `is_read`),
  INDEX `idx_expire` (`expire_time`),
  INDEX `idx_status` (`status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户邮件表';
```

### 6.2 具体关联设计

#### 与背包系统关联
- 邮件附件直接发放到背包
- 背包满时自动创建补发邮件
- 附件物品属性继承背包配置
- 背包容量检查在领取时执行

#### 与订单系统关联
- 充值成功后自动发送充值凭证邮件
- 月卡到期前发送续费提醒邮件
- 首充礼包通过邮件附件发放
- 订单异常时发送补偿邮件

#### 与活动系统关联
- 活动奖励自动通过邮件发放
- 活动结束前发送奖励提醒
- 限时活动邮件推送
- 活动专属道具通过邮件发放

#### 与任务系统关联
- 任务奖励大额发放通过邮件
- 成就奖励通知邮件
- 日常/周常奖励批量发放
- 任务未完成补偿邮件

#### 与客服系统关联
- 客服回复通过邮件发送
- 举报处理结果邮件通知
- 问题反馈自动回复
- GM指令执行结果通知
