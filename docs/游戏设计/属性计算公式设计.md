## 一、设计概述

### 1.1 公式体系架构

```
英雄最终属性 = 基础属性 × 品质系数 × (1 + 等级成长) × (1 + 星级加成) × (1 + 突破加成) + 宝物加成
```

### 1.2 属性类型说明

| 属性类型 | 说明 | 参与战力计算 |
|----------|------|-------------|
| HP (生命值) | 英雄存活能力 | 是 |
| Attack (攻击力) | 造成伤害的基础值 | 是 |
| Defense (防御力) | 减免受到的伤害 | 是 |
| Speed (速度) | 影响行动顺序 | 是 |

## 二、基础属性计算公式

### 2.1 英雄等级属性公式

英雄等级属性计算采用**分段指数增长**模式：

```
LevelAttr(level) = BaseAttr × (1 + GrowthRate × (level - 1) + QualityBonus × (level - 1)^1.2)
```

**简化版本（实际使用）：**

```
HeroAttr(level) = BaseAttr + Growth × (level - 1)
```

| 符号 | 说明 | 示例 |
|------|------|------|
| HeroAttr(level) | 某等级的基础属性 | - |
| BaseAttr | 1级时的基础属性 | 双钩定 HP=5000 |
| Growth | 每级成长值 | 双钩定 HP成长=150.5 |
| level | 当前等级 | 150 |

### 2.2 示例计算

**英雄：双钩定（SR品质）**
- 基础HP：5000
- HP成长：150.5

| 等级 | 计算过程 | HP结果 |
|------|----------|--------|
| 1 | 5000 + 150.5 × 0 | 5,000 |
| 50 | 5000 + 150.5 × 49 | 13,374 |
| 100 | 5000 + 150.5 × 99 | 19,899 |
| 150 | 5000 + 150.5 × 149 | 27,424 |
| 200 | 5000 + 150.5 × 199 | 34,949 |
| 300 | 5000 + 150.5 × 299 | 49,999 |

## 四、星级加成公式

### 4.1 星级系统说明

- 星级范围：1-6星
- 星级每提升1级，所有属性获得百分比加成
- 升星后属性**乘算**而非叠加

### 4.2 星级加成系数表

| 当前星级 | 目标星级 | HP加成 | 攻击加成 | 防御加成 | 速度加成 |
|----------|----------|--------|----------|----------|----------|
| 1 | 2 | +20% | +15% | +10% | +5% |
| 2 | 3 | +20% | +15% | +10% | +5% |
| 3 | 4 | +20% + | +15% | +10% | +5% |
| 4 | 5 | +20% | +15% | +10% | +5% |
| 5 | 6 | +20% | +15% | +10% | +5% |

### 4.3 星级累计加成公式

```
StarBonus(star) = ∏(i=2 to star) (1 + Bonus_i)
```

其中 Bonus_i 为第 i 星的单次加成。

**简化计算：**

| 星级 | 累计HP加成 | 累计攻击加成 | 累计防御加成 |
|------|------------|--------------|--------------|
| 1 | 100% | 100% | 100% |
| 2 | 120% | 115% | 110% |
| 3 | 144% | 132% | 121% |
| 4 | 173% | 152% | 133% |
| 5 | 207% | 175% | 146% |
| 6 | 249% | 201% | 161% |

### 4.4 星级加成示例

**英雄：双钩定（SR），150级**
- 基础HP：27,424
- 4星加成：173%（1.73）

```
HP(4星) = 27,424 × 1.73 = 47,443
```

## 五、突破加成公式

### 5.1 突破阶段说明

- 突破阶段：0-6阶
- 突破需满足等级要求
- 突破后属性获得百分比提升

### 5.2 突破属性加成表

| 阶段 | 等级要求 | HP加成 | 攻击加成 | 防御加成 | 速度加成 |
|------|----------|--------|----------|----------|----------|
| 0 | 1 | 0% | 0% | 0% | 0% |
| 1 | 30 | +10% | +5% | +5% | +2% |
| 2 | 70 | +10% | +5% | +5% | +2% |
| 3 | 120 | +15% | +10% | +10% | +5% |
| 4 | 180 | +15% | +10% | +10% | +5% |
| 5 | 250 | +20% | +15% | +15% | +10% |
| 6 | 300 | +20% | +15% | +15% | +10% |

### 5.3 突破累计加成公式

```
BreakthroughBonus(stage) = 1 + Σ(阶段加成百分比)
```

### 5.4 突破加成示例

**英雄：双钩定（SR），150级，3阶突破**
- 基础HP：27,424
- 1+2+3阶累计加成：10% + 10% + 15% = 35%

```
HP(3阶) = 27,424 × (1 + 0.35) = 37,022
```

## 六、宝物属性叠加公式

### 6.1 宝物属性类型

| 属性类型 | 说明 | 叠加方式 |
|----------|------|----------|
| 百分比属性 | attack_percent, hp_percent | 乘法叠加 |
| 固定值属性 | attack_fixed, hp_fixed | 加法叠加 |
| 特殊属性 | anger_damage, crit_damage | 加法叠加 |

### 6.2 宝物主属性公式

宝物主属性随等级成长：

```
ArtifactMainStat(level) = BaseValue + GrowthPerLevel × (level - 1)
```

### 6.3 宝物突破加成

```
ArtifactFinalStat = ArtifactMainStat(level) × (1 + BreakthroughBonus)
```

### 6.4 多宝物属性叠加

装备多个宝物时，同类型属性**加法叠加**：

```
TotalPercentAttr = Σ(各宝物百分比属性)
TotalFixedAttr = Σ(各宝物固定值属性)

FinalAttr = BaseAttr × (1 + TotalPercentAttr) + TotalFixedAttr
```

### 6.5 宝物加成示例

**英雄：双钩定（SR），150级，4星，3阶突破，装备2件宝物**

基础属性：
- HP：27,424

宝物属性：
- 发丘印（攻击+15%）
- 黑驴蹄子（HP+20%）

```
最终HP = 27,424 × (1 + 0.20) = 32,909
```

## 七、完整属性计算公式

### 7.1 最终属性公式

```
FinalAttr = (BaseAttr × QualityCoefficient + Growth × (level - 1)) 
            × StarMultiplier × BreakthroughMultiplier 
            × (1 + ArtifactPercentBonus) + ArtifactFixedBonus
```

### 7.2 参数说明

| 参数 | 说明 | 来源 |
|------|------|------|
| BaseAttr | 1级基础属性 | heroes表 |
| QualityCoefficient | 品质系数 | 品质系统 |
| Growth | 每级成长值 | heroes表 |
| level | 当前等级 | user_heroes表 |
| StarMultiplier | 星级累计加成 | 星级系统 |
| BreakthroughMultiplier | 突破累计加成 | 突破系统 |
| ArtifactPercentBonus | 宝物百分比加成 | 宝物系统 |
| ArtifactFixedBonus | 宝物固定值加成 | 宝物系统 |

### 7.3 完整示例计算

**条件：**
- 英雄：双钩定（SR品质=3）
- 等级：150
- 星级：4星
- 突破：3阶
- 宝物：黑驴蹄子（HP+20%）

**参数获取：**
- BaseHP = 5000
- QualityCoefficient = 1.35
- Growth = 150.5
- StarMultiplier(4星) = 1.73
- BreakthroughMultiplier(3阶) = 1.35
- ArtifactHPBonus = 0.20

**计算过程：**

```
1. 基础HP = 5000 × 1.35 + 150.5 × (150 - 1)
         = 6750 + 22424.5
         = 27,424

2. 星级加成 = 27,424 × 1.73 = 47,433

3. 突破加成 = 47,433 × 1.35 = 64,034

4. 宝物加成 = 64,034 × (1 + 0.20) = 76,841
```

**最终结果：HP = 76,841**

## 八、宝物升级属性公式

### 8.1 宝物主属性公式

宝物主属性提供百分比加成：

```
ArtifactMainStatValue = BaseValue + GrowthValue × (level - 1)
```

### 8.2 宝物副属性公式

副属性在突破后解锁并生效：

```
ArtifactSubStatValue = SubBaseValue + SubGrowthValue × breakthrough_stage
```

### 8.3 宝物品质系数

| 品质 | 主属性基础值 | 每级成长 |
|------|-------------|----------|
| 凡品(N) | 10% | 0.3% |
| 灵品(R) | 12% | 0.4% |
| 星品(SR) | 15% | 0.5% |
| 月品(SSR) | 18% | 0.6% |
| 日品(UR) | 22% | 0.8% |

### 8.4 宝物属性示例

**宝物：发丘印（SSR品质）**
- 主属性：攻击+18%
- 每级成长：0.6%
- 30级属性：18% + 0.6% × 29 = 35.4%
- 50级属性：18% + 0.6% × 49 = 47.4%

## 十、公式使用指南

### 10.1 前端计算

前端可使用简化公式进行UI展示，公式结果与服务器保持一致：

```javascript
// 伪代码
function calculateHeroHP(baseHP, quality, level, star, breakthrough, artifactBonus) {
  const qualityCoeff = QUALITY_COEFFICIENT[quality];
  const starMultiplier = calculateStarMultiplier(star);
  const breakthroughMultiplier = calculateBreakthroughMultiplier(breakthrough);
  
  const baseValue = baseHP * qualityCoeff + getGrowthHP() * (level - 1);
  const afterStar = baseValue * starMultiplier;
  const afterBreakthrough = afterStar * breakthroughMultiplier;
  const finalHP = afterBreakthrough * (1 + artifactBonus);
  
  return Math.floor(finalHP);
}
```

### 10.2 数值平衡建议

1. **成长曲线**：建议成长曲线在后期略有放缓，避免属性爆炸
2. **品质差异**：UR与N的品质差异应控制在2-3倍
3. **星级收益**：高星级的属性收益应明显高于低星级
4. **突破收益**：突破应提供显著的属性提升，体现投入价值
