# 签到系统设计文档

## 一、系统概述

### 1.1 系统定位
签到系统是《苍穹劫·摸金传人》的核心日常活跃系统，通过每日签到奖励激励玩家保持登录习惯，建立稳定的用户活跃循环。系统设计兼顾免费玩家与付费玩家的需求，通过连续签到奖励和补签机制提升用户粘性。

### 1.2 设计目标
- **日活提升**：通过签到奖励确保每日登录
- **连续激励**：通过连续签到奖励鼓励持续登录
- **付费转化**：通过补签卡等道具引导付费
- **情感维系**：通过签到故事和奖励提升游戏体验

### 1.3 设计原则
- **简洁直观**：签到操作简单，一键完成
- **奖励明确**：奖励展示清晰，所见即所得
- **连续价值**：突出连续签到的额外价值
- **补签兜底**：给非连续签到玩家补救机会

```
├── 每4天触发一次连续签到奖励
├── 连续签到奖励为当日基础奖励的1.5-2倍
└── 连续签到中断则重新计算

额外奖励：
├── 7天连续：额外赠送稀有召唤券×1
├── 14天连续：额外赠送史诗召唤券×1
├── 21天连续：额外赠送限定头像框
└── 28天连续：额外赠送限定称号+橙色武器

断签处理：
├── 使用补签卡可补签前一天
├── 补签不中断连续签到状态
└── 每月最多补签3次
```

### 2.3 补签功能设计

#### 2.3.1 补签规则
| 补签类型 | 获取方式 | 每月限制 | 价格 |
|---------|---------|---------|------|
| 免费补签卡 | 活动发放 | 1次/月 | 免费 |
| 月卡补签卡 | 月卡用户 | 2次/月 | 含在月卡内 |
| 付费补签卡 | 商城购买 | 无限 | 30钻石/次 |

#### 2.3.2 补签流程
```
补签操作流程：

1. 玩家未签到某一天
2. 次日登录弹出补签提示
3. 玩家选择使用补签卡
4. 系统扣除补签卡
5. 自动完成昨日签到
6. 补签记录存档
7. 连续签到状态保持
```

### 5.1 签到奖励配置表 (signin_rewards)

```sql
CREATE TABLE `signin_rewards` (
  `id` INT AUTO_INCREMENT PRIMARY KEY,
  `day` INT NOT NULL COMMENT '签到天数1-30',
  `cycle_type` VARCHAR(16) DEFAULT 'monthly' COMMENT '周期类型',
  `reward_type` VARCHAR(16) DEFAULT 'normal' COMMENT '奖励类型：normal/consecutive/vip',
  `reward_gold` INT DEFAULT 0 COMMENT '金币奖励',
  `reward_diamond` INT DEFAULT 0 COMMENT '钻石奖励',
  `reward_item_id` VARCHAR(32) COMMENT '道具奖励ID',
  `reward_item_count` INT DEFAULT 0 COMMENT '道具数量',
  `reward_item2_id` VARCHAR(32) COMMENT '第二道具ID',
  `reward_item2_count` INT DEFAULT 0 COMMENT '第二道具数量',
  `is_consecutive_day` TINYINT DEFAULT 0 COMMENT '是否连续奖励日',
```

### 6.2 具体关联设计

#### 与背包系统关联
- 签到道具直接发放到背包
- 背包满时道具自动发送到邮件
- 背包空间检查在奖励发放前执行
- 背包容量提示引导购买月卡

#### 与邮件系统关联
- 背包满时奖励通过邮件发放
- 补签奖励通过邮件附件发送
- 连续签到达标通知邮件
- 签到奖励领取状态邮件提醒

#### 与商城系统关联
- 补签卡在商城有售
- 签到奖励引导商城消费
- 首充礼包与签到联动
- 商城消费可获额外补签机会

#### 与月卡系统关联
- 月卡用户获得额外补签次数
- 月卡用户签到奖励翻倍
- 月卡用户专属签到奖励
- 月卡续费与签到连续中断提示

#### 与任务系统关联
- 每日签到计入日常任务
- 连续签到计入成就任务
- 签到任务可获得额外补签卡
- 签到活跃度影响任务奖励

#### 与活动系统关联
- 节假日签到奖励倍增
- 签到活动期间奖励翻倍
- 限时签到活动专属奖励
- 签到排行榜活动

## 七、附录

### 7.1 签到时间规则
```yaml
时间规则:
  服务器时间:
    - 所有签到以服务器时间为准
    - 支持配置不同时区
    
  签到窗口:
    - 每日 00:00 - 23:59 可签到
    - 次日 00:00 前可补签昨日
    
  周期重置:
    - 每月1日 00:00 重置
    - 重置后上月数据归档
```

### 7.2 防刷机制
```yaml
安全机制:
  1. IP限制
     - 同一IP每日签到上限
     - 异常IP标记和限制
  
  2. 设备限制
     - 同一设备签到账号限制
     - 设备指纹识别
  
  3. 行为检测
     - 异常签到时间检测
     - 批量操作检测
  
  4. 数据校验
     - 签到数据服务器校验
     - 奖励发放二次确认
```

### 7.3 数据统计指标
```yaml
核心指标:
  1. 活跃指标
     - 日签到率 = 签到人数 / 活跃人数
     - 连续7天签到率
     - 月度满签率
  
  2. 奖励指标
     - 奖励领取率
     - 补签使用率
     - 道具消耗率
  
  3. 转化指标
     - 签到→付费转化
     - 补签→付费转化
     - 签到→留存影响
```
