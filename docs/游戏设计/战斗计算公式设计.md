# 《苍穹劫·摸金传人》战斗计算公式设计

> 计算公式文档 - 版本 1.0

## 一、设计概述

### 1.1 战斗公式体系

```
伤害输出 = (攻击方攻击 - 防御方防御 × 防御减免) × 暴击加成 × 伤害浮动 × 技能系数 × (1 + 伤害加成 - 伤害减免)
```

### 1.2 战斗属性分类

| 属性类型 | 说明 | 作用 |
|----------|------|------|
| 基础属性 | HP/ATK/DEF/SPD | 核心战斗数值 |
| 攻击属性 | 暴击率/暴击伤害/命中 | 提升输出 |
| 防御属性 | 闪避率/抗暴率/韧性 | 减少受击伤害 |
| 特殊属性 | 伤害加成/伤害减免 | 最终伤害调节 |

## 二、基础伤害公式

### 2.1 最终伤害公式

```
Damage = BaseDamage × (1 + DamageBonus - DamageReduction) × CriticalBonus × FloatBonus
```

### 2.2 基础伤害计算

```
BaseDamage = (ATK × SkillCoeff - DEF × 0.5) × (ATK / (ATK + DEF × 2))
```

**简化版本：**

```
BaseDamage = ATK × SkillCoeff × (1 - DEF / (DEF + ATK × 2))
```

### 2.3 防御减免公式

防御力对伤害的减免遵循**递减曲线**：

```
DefenseReduction = DEF / (DEF + BaseConstant)
```

- BaseConstant = 1000（防御常量）

### 2.4 防御减免示例

| 防御力 | 防御减免 | 有效减伤 |
|--------|----------|----------|
| 0 | 0% | 0% |
| 200 | 16.7% | 16.7% |
| 500 | 33.3% | 33.3% |
| 1000 | 50.0% | 50.0% |
| 2000 | 66.7% | 66.7% |
| 5000 | 83.3% | 83.3% |
| 10000 | 90.9% | 90.9% |

### 2.5 伤害计算完整示例

**攻击方：双钩定**
- ATK = 5000
- 技能系数 = 150%

**防御方：古墓守卫**
- DEF = 1000

**计算过程：**

```
1. 防御减免 = 1000 / (1000 + 1000) = 50%

2. BaseDamage = 5000 × 1.5 × (1 - 0.5) = 3750

3. 最终伤害 = 3750 × (1 + 0 - 0) × 1.0 × 1.0 = 3750
```

## 三、暴击公式

### 3.1 暴击判定

暴击触发基于随机数判定：

```
IsCritical = Random(0, 10000) < CriticalRate × 10000
```

- CriticalRate：暴击率（范围 0-1）
- 精度：小数点后4位

### 3.2 暴击伤害公式

```
CriticalBonus = 1 + CriticalDamageBonus
```

| 状态 | 伤害倍率 |
|------|----------|
| 普通攻击 | 1.0 (100%) |
| 暴击攻击 | 1.5~2.0 (150%~200%) |

### 3.3 暴击属性上限

| 属性 | 软上限 | 硬上限 |
|------|--------|--------|
| 暴击率 | 80% | 100% |
| 暴击伤害 | 200% | 250% |

### 3.4 暴击相关属性

| 属性名称 | 说明 | 默认值 |
|----------|------|--------|
| crit_rate | 暴击率 | 5% |
| crit_damage | 暴击伤害 | 150% |
| anti_crit_rate | 抗暴率 | 0% |
| crit_damage_reduction | 暴击伤害减免 | 0% |

### 3.5 暴击公式应用

```
EffectiveCritDamage = BaseCritDamage + CritDamageBonus - CritDamageReduction
```

## 四、命中与闪避公式

### 4.1 命中判定

```
HitChance = HitRate / (HitRate + EvasionRate × 2 + 50)
```

### 4.2 闪避判定

```
EvasionChance = EvasionRate / (HitRate + EvasionRate × 2 + 50)
```

### 4.3 基础命中/闪避

| 属性 | 基础值 | 说明 |
|------|--------|------|
| HitRate | 100% | 基础命中率 |
| EvasionRate | 0% | 基础闪避率 |

### 4.4 命中闪避上限

| 属性 | 上限 |
|------|------|
| 命中率 | 100% |
| 闪避率 | 80% |

### 4.5 命中判定示例

**攻击方命中率：100%**
**防御方闪避率：30%**

```
HitChance = 100 / (100 + 30 × 2 + 50) = 100 / 210 = 47.6%
```

## 六、怒气生成公式

### 6.1 怒气基础规则

- 怒气上限：100点
- 普攻获得怒气：10-15点
- 受伤获得怒气：5-10点
- 击杀目标获得怒气：20点

### 6.2 怒气获取公式

```
AngerGain = BaseAnger + LevelBonus × (1 + BonusRate)
```

### 6.3 怒气获取配置

| 触发条件 | 基础怒气 | 等级修正 |
|----------|----------|----------|
| 普攻命中 | 10 | +0.5/级 |
| 怒攻释放 | 0 | - |
| 受到普攻 | 5 | +0.3/级 |
| 受到怒攻 | 10 | +0.5/级 |
| 击杀目标 | 20 | +1/级 |
| 回合结束 | 3 | - |

### 6.4 怒气回复速度

```
AngerPerSecond = BaseAngerRegen × (1 + AngerRegenBonus)
```

- BaseAngerRegen = 0（放置战斗中自动回复）

### 6.5 怒气机制说明

| 阶段 | 怒气需求 | 释放时机 |
|------|----------|----------|
| 常规战斗 | 100 | 可主动/自动释放 |
| 放置收益 | 100 | 自动释放 |

## 七、速度与行动顺序

### 7.1 速度计算

```
EffectiveSpeed = BaseSpeed × (1 + SpeedBonus)
```

### 7.2 行动顺序公式

战斗开始时，按速度决定行动顺序：

```
TurnOrder = SortDesc(EffectiveSpeed)
```

### 7.3 速度属性配置

| 品质 | 基础速度 | 速度成长 |
|------|----------|----------|
| N | 100 | 2.0 |
| R | 105 | 2.2 |
| SR | 110 | 2.5 |
| SSR | 115 | 2.8 |
| UR | 120 | 3.0 |

### 7.4 速度示例

**双钩定（SR），150级**
- BaseSpeed = 110
- SpeedGrowth = 2.5
- 速度成长 = 110 + 2.5 × (150 - 1) = 110 + 372.5 = 482.5

### 7.5 回合行动计算

每回合行动后，下次行动时间：

```
NextActionTime = CurrentTime + (BaseTickTime / EffectiveSpeed) × 10000
```

- BaseTickTime = 10000（毫秒）

## 八、伤害加成与减免

### 8.1 伤害加成公式

```
FinalDamageBonus = Σ(DamageBonus_i) - Σ(DamageReduction_i)
```

### 8.2 伤害加成类型

| 加成类型 | 来源 | 上限 |
|----------|------|------|
| 对 Boss 伤害 | 装备/技能 | +100% |
| 对玩家伤害 | 装备/技能 | +50% |
| 普攻伤害 | 被动技能 | +30% |
| 怒攻伤害 | 突破解锁 | +50% |
| 属性伤害 | 装备词缀 | +50% |

### 8.3 伤害减免类型

| 减免类型 | 来源 | 上限 |
|----------|------|------|
| 受到伤害 | 被动技能 | -30% |
| 受到普攻 | 装备/技能 | -20% |
| 受到怒攻 | 装备/技能 | -20% |
| 属性伤害减免 | 装备词缀 | -50% |

### 8.4 最终伤害公式（完整）

```
FinalDamage = BaseDamage 
              × (1 + DamageBonus - DamageReduction)
              × CriticalMultiplier
              × FloatMultiplier
              × ElementAdvantage
              × TypeAdvantage
```

## 九、完整战斗计算示例

### 9.1 战斗场景

**攻击方：双钩定（SR）**
- 等级：150
- ATK：5000
- 暴击率：30%
- 暴击伤害：180%
- 技能系数：150%

**防御方：古墓粽子（精英）**
- 等级：150
- DEF：2000
- 伤害减免：10%

### 9.2 普攻计算

```
1. 防御减免 = 2000 / (2000 + 1000) = 66.7%

2. BaseDamage = 5000 × 1.0 × (1 - 0.667) = 1667

3. 伤害加成 = 0%

4. 最终伤害 = 1667 × (1 + 0) = 1667
```

### 9.3 怒攻计算（暴击）

```
1. 防御减免 = 2000 / (2000 + 1000) = 66.7%

2. BaseDamage = 5000 × 1.5 × (1 - 0.667) = 2500

3. 暴击判定：30% < Random(0-10000)，假设触发暴击

4. 暴击伤害 = 1 + 0.80 = 1.8

5. 伤害浮动 = 105%（随机取最大值）

6. 最终伤害 = 2500 × 1.8 × 1.05 = 4725
```

## 十一、战斗公式使用指南

### 11.1 前端展示计算

```javascript
function calculateDamage(attacker, defender, skillCoeff) {
  const baseDamage = attacker.atk * skillCoeff * 
    (1 - defender.def / (defender.def + 1000));
  
  const afterBonus = baseDamage * (1 + attacker.damageBonus - defender.damageReduction);
  
  const isCrit = Math.random() < attacker.critRate;
  const critMultiplier = isCrit ? (1 + attacker.critDamage) : 1;
  
  const floatMultiplier = (950 + Math.random() * 150) / 1000;
  
  return Math.floor(baseDamage * afterBonus * critMultiplier * floatMultiplier);
}
```

### 11.2 战斗平衡建议

1. **防御收益递减**：确保防御不会完全抵消伤害
2. **暴击价值**：暴击伤害应高于普通伤害的50%以上
3. **速度价值**：速度优势应体现为先手优势
4. **怒气节奏**：确保每场战斗能释放2-3次怒攻
