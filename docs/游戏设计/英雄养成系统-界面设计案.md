# 《苍穹劫》英雄养成系统 界面设计方案

> 版本：v1.0
> 日期：2026年4月23日
> 范围：英雄养成主界面、英雄详情界面、升级/突破流程、宝物系统

---

## 一、设计原则

### 1.1 整体风格

- **风格定位**：古墓探险暗黑风 + 国风武侠元素
- **色彩体系**：
  - 主色：深青铜色 `#8B7355`
  - 强调色：古铜金 `#B8860B`
  - 背景：墓室深灰 `#1A1A1A`
  - 文字：羊皮纸黄 `#F5DEB3`
  - 突破特效：玄光紫 `#9B59B6`
  - 升星特效：流光金 `#FFD700`

### 1.2 布局原则

- **手机竖屏优先**：适配 9:16 ~ 9:20 屏幕比例
- **信息层级**：核心数据显眼，次要信息可折叠
- **一键操作**：减少操作步骤，常用功能一步到位
- **反馈明确**：每个操作都有视觉/数值反馈

### 2.2 英雄卡片设计

**英雄卡片内容**：

| 区域 | 内容 | 说明 |
|------|------|------|
| 立绘区域 | 英雄立绘（半透明遮罩） | 品质决定边框颜色 |
| 品质边框 | N灰/R绿/SR蓝/SSR紫/UR金 | 边框发光效果 |
| 等级显示 | `Lv.150` 大字 | 右下角金色数字 |
| 星级显示 | ★★★☆☆ | 右上角，灰色填充表示未升满 |
| 突破标记 | ⓸ | 突破阶段，左上角 |
| 宝物图标 | 小图标 | 底部中间，有装备显示图标 |
| 快捷操作 | [升级] [突破] | 底部按钮 |

**卡片尺寸**：宽度 48% 屏幕宽度，高度 200px

**交互状态**：

| 状态 | 视觉效果 |
|------|---------|
| 默认 | 静态立绘 + 边框微光 |
| 可升级 | 等级数字呼吸动画（绿色箭头↑） |
| 可突破 | 突破标记脉冲发光（紫色） |
| 选中 | 放大1.05 + 阴影加深 |
| 锁定 | 灰度处理 + 锁图标 |

### 2.3 排序与筛选

**排序选项**（点击顶部排序按钮展开）：

| 排序方式 | 图标 | 说明 |
|---------|------|------|
| 等级高→低 | ↑↓ | 默认 |
| 星级高→低 | ★ | 按星级排序 |
| 品质高→低 | ◆ | SSR > SR > R > N |
| 突破高→低 | ⓪ | 按突破阶段 |
| 入手顺序 | 🕐 | 最新获得优先 |

**筛选标签**：

- `[全部]` - 显示所有英雄
- `[R]` `[SR]` `[SSR]` `[UR]` - 按稀有度筛选
- 可多选，选中状态为实心

### 2.4 小队等级进度条

**位置**：英雄列表下方，固定显示

**显示内容**：

```
┌─────────────────────────────────────────┐
│  小队等级: Lv.35                        │
│  ████████████░░░░░░░░░░░░  45%         │
│  下一级还需: 2,500 / 5,500 经验         │
└─────────────────────────────────────────┘
```

**小队等级提升**：
- 所有英雄总等级 / 英雄数量 = 小队等级
- 升级时播放全屏金色光芒动画

## 四、升级/突破流程

### 4.1 升级流程

**入口**：详情界面 [升级] 按钮 或 英雄卡片快捷升级

**交互流程**：

```
步骤1: 点击[升级]
         ↓
┌─────────────────────────┐
│  升级确认               │
│  ┌───────────────────┐  │
│  │ Lv.150 → Lv.151  │  │
│  │ 消耗: 经验药水(中) │  │
│  │        x3 (已有x15)│  │
│  │ 获得: +50 生命    │  │
│  │       +8  攻击    │  │
│  │       +3  防御    │  │
│  └───────────────────┘  │
│                         │
│  [自动] [手动+1] [手动+5]│  升级模式选择
│                         │
│  [取消]     [确认升级]   │
└─────────────────────────┘
         ↓ (点击确认)
      ┌─────────┐
      │ +1 Lv.  │  飘字动画 (+50HP ↑)
      │ +1 Lv.  │
      │ +1 Lv.  │
      └─────────┘
         ↓ (播放完毕)
      属性条+数值更新动画
```

**动画效果**：
- 升级数字飘字：`+1 Lv.` 从卡片中心向上飘出并消失
- 属性条增长：从左向右填充，带金色粒子
- 升级完成：全屏短暂金色闪光

## 七、数据结构概览

### 7.1 界面数据依赖

```
英雄养成主界面
├── heroList: Hero[]
│   ├── id, name, rarity, level, starLevel, breakLevel
│   └── equippedTreasureId
├── teamLevel: number (小队等级)
├── teamExp: number / teamExpForNext
└── sortType, filterRarity[]

英雄详情界面
├── hero: Hero
├── heroData: HeroCultivationData
│   ├── level, exp, breakLevel, starLevel, rage
│   └── equippedTreasureId
├── finalStats: Stats (计算后)
├── skillDetails: Skill[]
└── treasure: Treasure (如果已装备)

宝物背包界面
├── treasureList: Treasure[]
│   ├── id, name, type, quality, level, breakLevel
│   ├── mainAttr, subAttr, specialEffect
│   └── ownerHeroId (null表示未装备)
└── filterType, filterQuality[]

宝物详情界面
├── treasure: Treasure
├── strengtheningCost: Material[]
├── breakCost: Material[]
└── previewStats: Stats (强化/突破后预览)
```

### 7.2 关键计算公式（前端展示用）

```javascript
// 小队等级 = 所有英雄等级之和 / 英雄数量
teamLevel = sum(hero.level for hero in heroList) / heroList.length

// 星级属性加成
starBonus = { 1: 1.0, 2: 1.2, 3: 1.4, 4: 1.6, 5: 1.8, 6: 2.2 }

// 突破等级加成
breakBonus = 1 + (breakLevel * 0.05)

// 品质倍率
rarityMultiplier = { N: 0.7, R: 0.85, SR: 1.0, SSR: 1.3, UR: 1.5 }

// 宝物属性系数
treasureQualityMultiplier = { N: 0.7, R: 0.85, SR: 1.0, SSR: 1.2, UR: 1.5 }

// 最终属性
finalHP = (baseHP + hpGrowth * (level-1)) * starBonus * breakBonus * rarityMultiplier
```
