## 一、设计概述

### 1.1 活动系统架构

```
活动系统
├── 活动配置（活动定义）
├── 活动奖励（奖励配置）
├── 玩家进度（参与记录）
└── 排行榜（排名系统）
```

| 序号 | 表名 | 说明 |

| 1 | activities | 活动配置表 |

## 八、常用查询示例

### 8.1 查询进行中活动

```sql
SELECT a.*, 
       (SELECT COUNT(*) FROM user_activity_progress WHERE activity_id = a.id) as participant_count
FROM activities a
WHERE a.is_active = 1 
    AND a.start_time <= NOW() 
    AND a.end_time >= NOW()
ORDER BY a.priority DESC, a.sort_order;
```

### 8.2 查询玩家活动状态

```sql
SELECT 
    a.id as activity_id,
    a.activity_name,
    a.activity_type,
    uap.current_value,
    uap.claim_status,
    CASE WHEN uap.id IS NULL THEN 0 ELSE 1 END as is_participated
FROM activities a
LEFT JOIN user_activity_progress uap ON a.id = uap.activity_id AND uap.user_id = ?
WHERE a.is_active = 1 
    AND a.start_time <= NOW() 
    AND a.end_time >= NOW()
ORDER BY a.priority DESC;
```

### 8.3 查询活动奖励领取状态

```sql
SELECT 
    ar.id as reward_id,
    ar.reward_name,
    ar.items_desc,
    ar.condition_value,
    COALESCE(uap.claim_status->>CONCAT('$.', ar.condition_value), '0') as claimed
FROM activity_rewards ar
JOIN activities a ON ar.activity_id = a.id
LEFT JOIN user_activity_progress uap ON a.id = uap.activity_id AND uap.user_id = ?
WHERE a.id = ? AND ar.is_active = 1
ORDER BY ar.condition_value;
```

### 8.4 查询排行榜

```sql
SELECT 
    ar.rank,
    ar.user_name,
    ar.score,
    ar.rank_change,
    CASE WHEN ar.rank <= 10 THEN 1 ELSE 0 END as show_avatar
FROM activity_rankings ar
WHERE ar.activity_id = ?
ORDER BY ar.rank
LIMIT 100;
```
