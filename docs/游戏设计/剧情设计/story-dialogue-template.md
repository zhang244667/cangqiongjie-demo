# 《苍穹劫·摸金传人》对白脚本模板

## 一、对白格式规范

### 1.1 基础格式

```
【场景】：场景描述（自动生成或手动指定）
【角色头像】：显示对应角色立绘
【角色名】：角色名称
━━━━━━━━━━━━━━━━━━━━━━━━
对白内容（自动播放或玩家点击继续）
━━━━━━━━━━━━━━━━━━━━━━━━
【选项】（如有分支对话）
　A. 选项A内容
　B. 选项B内容
　C. 选项C内容
```

### 1.2 格式示例

```markdown
【场景】：苍龙七宿墓·角宿墓入口。夜色如墨，残月隐于云后。
【陈星垣】：（观察四周）这里就是爷爷提到的古墓入口......角宿之位。

【陈星垣】：（取出半张星图）地图指向这里，应该没错。

【陈星垣】：（自言自语）爷爷......你到底想让我在这里找到什么？

━━━━━━━━━━━━━━━━━━━━━━━━
【选项】
　A. "先进去看看再说。"
　B. "等一下，观察一下周围环境。"
　C. "云绮罗，你怎么看？"
```

### 1.3 对白结构标记

| 标记 | 含义 | 用途 |
|------|------|------|
| `【场景】` | 场景描述 | 交代环境、地点、时间 |
| `【角色名】` | 说话角色 | 明确谁在说话 |
| `（动作）` | 角色动作 | 表情、动作、行为描写 |
| `（旁白）` | 旁白描述 | 场景变化、氛围描写 |
| `【选项】` | 玩家选择 | 分支对话入口 |
| `━━━━━━━━━━━━━━━━` | 分隔线 | 对话段落分隔 |
| `★` | 重点标记 | 关键信息提示 |
| `◆` | 道具标记 | 获得物品提示 |
| `※` | 警告标记 | 危险/诅咒提示 |

### 1.4 对白类型分类

| 类型 | 标识 | 说明 |
|------|------|------|
| 剧情对话 | 【剧情】 | 推进主线剧情，不可跳过 |
| 探索对话 | 【探索】 | 可自由对话，交互内容 |
| 战斗对话 | 【战斗】 | Boss战或普通战斗 |
| 过场对话 | 【过场】 | 纯播片，可快进 |
| 结局对话 | 【结局】 | 结局分支选择 |

## 二、对白触发时机

## 三、分支对话设计

### 3.1 分支类型

| 类型 | 说明 | 玩家感受 |
|------|------|----------|
| **影响分支** | 选项影响后续剧情走向 | 有掌控感 |
| **信息分支** | 不同选项获得不同信息 | 有探索感 |
| **关系分支** | 选项影响角色好感度 | 有代入感 |
| **隐藏分支** | 特定条件解锁隐藏选项 | 有惊喜感 |

### 3.2 分支对话示例

#### 示例1：影响分支

```markdown
【场景】：紫微玄穹墓入口处，陈星垣面对分岔路口。

【陈星垣】：左边是安全的常规通道，右边有一条隐蔽小道......

【萧破军】：老萧觉得走大路稳妥点。

【云绮罗】：我感觉右边......有什么在呼唤我。

【石敢当】：......

━━━━━━━━━━━━━━━━━━━━━━━━
【选项】
　A. "走大路，稳妥为上。" → 大路路线，常规奖励
　B. "走小道，云绮罗你感应到什么？" → 小道路线，隐藏剧情
　C. "分头行动，一队走一边。" → 分队模式，高风险高回报
```

#### 示例2：信息分支

```markdown
【场景】：墓室深处发现一块古碑，碑文模糊难辨。

【陈星垣】：这碑文......是上古星文，需要解读。

【云绮罗】：让我来试试。（开始解读）

━━━━━━━━━━━━━━━━━━━━━━━━
【选项】
　A. "解读碑文内容" → 获得完整历史信息
　B. "先记录下来，以后再说" → 获得部分信息，跳过解读
　C. "仔细观察石碑本身" → 发现石碑隐藏的机关
```

#### 示例3：关系分支

```markdown
【场景】：营地休息，云绮罗独自坐在一旁。

【云绮罗】：（望着星空）今晚的星星......和母亲离开那天一样美。

【陈星垣】：......

━━━━━━━━━━━━━━━━━━━━━━━━
【选项】
　A. "你母亲......是怎么去世的？"（+5好感）→ 触发云绮罗身世剧情
　B. "别想太多，早点休息吧。"（+2好感）→ 温和安慰
　C. "（沉默，在她身边坐下）"（+3好感）→ 陪伴无言
```

#### 示例4：隐藏分支

```markdown
【场景】：发现一处隐蔽角落，地上有微弱光芒。

【陈星垣】：这里......有东西。

【云绮罗】：是暗门！

※系统提示：发现隐藏区域！好感度≥60时解锁

━━━━━━━━━━━━━━━━━━━━━━━━
【选项】（隐藏条件满足）
　★ A. "让我仔细看看这个光。"（触发隐藏剧情）→ 发现陈家先祖遗物
　A. "打开这个门。" → 进入隐藏区域
　B. "先标记位置，以后再来。" → 保存进度
```

## 四、情感表达指南

### 4.1 情感类型定义

| 情感类型 | 场景 | 表达方式 | 示例语气词 |
|----------|------|----------|------------|
| **平静** | 日常对话 | 平稳、自然 | "嗯。" "好。" "走吧。" |
| **喜悦** | 获得道具/胜利 | 轻快、高昂 | "太好了！" "成功了！" |
| **悲伤** | 回忆/失败 | 低沉、缓慢 | "......" "是吗......" |
| **愤怒** | 被激怒/背叛 | 急促、有力 | "混账！" "可恶！" |
| **恐惧** | 遭遇危险 | 颤抖、停顿 | "这......这是......" |
| **惊讶** | 发现真相 | 突然、加重 | "什么？！" "不可能！" |
| **坚定** | 下定决心 | 沉稳、有力 | "......我明白了。" |
| **疑惑** | 遇到谜题 | 疑问、思考 | "这是什么意思......" |
| **绝望** | 诅咒发作/失败 | 虚弱、压抑 | "......没用的......" |

### 4.2 情感表现技巧

#### 4.2.1 语速变化

```markdown
【平静对话】
【陈星垣】：这里的构造......和爷爷说的一样。（语速：正常）

【惊讶对话】
【陈星垣】：什......什么？！爷爷他还活着？！ （语速：急促，重复）

【悲伤对话】
【陈星垣】：......原来是这样。（语速：缓慢，停顿）

【愤怒对话】
【陈星垣】：你们......竟敢动这个地方！（语速：加重，有力）
```

#### 4.2.2 动作描写

```markdown
【紧张】：（后退一步，手握罗盘）
【坚定】：（握紧拳头，眉心星纹微亮）
【恐惧】：（浑身发抖，冷汗直流）
【悲伤】：（低下头，声音颤抖）
【愤怒】：（眼中闪过寒光，周身气压骤降）
【虚弱】：（扶住墙壁，摇摇欲坠）
```

#### 4.2.3 表情标记

| 标记 | 表情 | 使用场景 |
|------|------|----------|
| `(◕‿◕)` | 微笑 | 日常、温和 |
| `(′▽′)` | 开心 | 喜悦、成功 |
| `(´_\`)` | 无奈 | 无奈、苦笑 |
| `(′⊙ω⊙` | 惊讶 | 震惊 |
| `╭(╯^╰)╮` | 坚定 | 决心 |
| `(っ˘̩╭╮˘̩)っ` | 担心 | 关心 |
| `(ง •̀_•́)ง` | 战斗 | 战斗准备 |
| `(╥﹏╥)` | 哭泣 | 悲伤 |

#### 4.2.4 音效配合

| 情感 | 音效建议 | 时长 |
|------|----------|------|
| 平静 | 风声、环境音 | 持续 |
| 喜悦 | 轻快铃声 | 0.5s |
| 悲伤 | 低沉弦乐 | 持续 |
| 愤怒 | 雷鸣、鼓声 | 1s |
| 恐惧 | 诡异风声、低语 | 持续 |
| 战斗 | 战斗BGM切换 | 即时 |
| 诅咒 | 刺耳杂音、符文声 | 持续至结束 |

## 八、对白脚本编写规范

### 8.1 编写原则

1. **简洁性**：每句对白不超过50字
2. **角色一致性**：每个角色的说话风格保持统一
3. **信息密度**：关键信息必须清晰传递
4. **情感真实**：符合角色性格和当前情境
5. **可执行性**：每条对白都要有明确的触发和结果

### 8.2 格式检查清单

```markdown
✓ 对话ID：chapterX_sceneY_dialogueZ
✓ 触发条件：明确
✓ 角色状态：标注（如受伤/诅咒发作）
✓ 场景描述：完整
✓ 对话内容：符合角色性格
✓ 选项设计：合理，有区分度
✓ 结果记录：明确记录选择后果
✓ 音频配置：音效/语音标注
✓ 动画配置：表情/动作标注
```

### 8.3 质量评估标准

| 评分项 | 优秀（5分） | 良好（4分） | 合格（3分） | 不合格（<3分） |
|--------|-------------|-------------|-------------|----------------|
| 角色一致性 | 完全符合 | 基本符合 | 部分符合 | 偏离角色 |
| 信息传递 | 清晰完整 | 基本清晰 | 有遗漏 | 信息混乱 |
| 情感表达 | 精准动人 | 较为准确 | 一般 | 情感缺失 |
| 交互设计 | 丰富合理 | 基本合理 | 选择单一 | 无分支 |
| 可执行性 | 完全可执行 | 95%可执行 | 80%可执行 | 需要大量修改 |
