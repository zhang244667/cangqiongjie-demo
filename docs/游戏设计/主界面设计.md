# 主界面设计文档

## 一、设计理念

采用**场景交互式**方案，让玩家感觉不是在点按钮，而是在一个真实的盗墓营地中探索。

---

## 二、核心入口布局

### 2.1 布局示意图

```
┌─────────────────────────┐
│  [顶部资源栏]            │
├─────────────────────────┤
│                         │
│      [公会旗帜]          │  ← 后方，次要
│                         │
│  [休息帐篷]   [星盘祭坛]  │  ← 角色 / 召唤
│     (左)        (右)     │
│                         │
│     [古墓入口/通道]       │  ← 核心入口，最显眼
│        (中央)            │
│                         │
│  [古董摊]      [告示板]   │  ← 商城 / 任务
│   (左下)       (右下)     │
├─────────────────────────┤
│ [角色] [编队] [背包] [设置] │
└─────────────────────────┘
```

### 2.2 核心原则

1. **探险入口最重要** → 放中央最显眼位置
2. **视觉层次分明** → 通过位置、大小、光效区分优先级
3. **场景融合** → 用场景元素代替传统按钮，更有沉浸感

## 六、场景氛围方案

### 方案A：神秘营地（推荐）
- 时间：黄昏
- 光源：灯笼暖光
- 氛围：神秘、温暖、有生活气息
- 适合：通用，平衡感好

### 方案B：阴暗古风
- 时间：月夜
- 光源：石灯微光 + 雾气
- 氛围：阴暗、紧张、恐怖感
- 适合：硬核盗墓氛围

### 方案C：星夜营地（世界观强化）
- 时间：星夜
- 元素：星盘图案 + 天空裂痕
- 氛围：史诗、玄幻
- 适合：突出"古墓即星盘"概念

## 七、关卡内界面布局（game.html）

### 7.1 三层结构

```
┌─────────────────────────┐
│      顶部状态栏         │  ← 火把/生命/金币/机关进度
├─────────────────────────┤

├─────────────────────────┤

└─────────────────────────┘
```

### 7.2 顶部状态栏

```
┌─────────────────────────────────────┐
│ 🔥火把5  ❤️78%  🪙320  📊机关3/8   │
└─────────────────────────────────────┘
```

| 信息 | 显示方式 |
|------|----------|
| 火把 | 数量 + 图标 |


| 按钮 | 功能 | 打开方式 |
|------|------|----------|
| 👥 队伍 | 查看/管理队伍 | 底部抽屉 |
| 🎒 背包 | 查看物品 | 底部抽屉 |
| 📜 日志 | 探索记录 | 全屏页面 |
| ⚙️ 设置 | 游戏设置 | 弹窗 |

## 八、交互方式

### 8.1 入口交互
- 点击场景元素 → 打开对应功能
- 入口hover效果 → 轻微缩放 + 光效增强
- 未开放入口 → 点击提示"开发中"

### 8.2 动画效果
- 探险入口：金色光晕呼吸动画
- 召唤入口：紫色光效脉动 + 浮动粒子
- 其他入口：温暖灯光微动
