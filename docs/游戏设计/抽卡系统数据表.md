## 一、设计概述

### 1.1 抽卡系统架构

```
抽卡系统
├── 抽卡池配置（奖池定义）

├── 抽卡记录（历史追溯）

```

## 七、保底机制设计

### 7.1 保底规则配置

| 池类型 | 保底品质 | 保底阈值 | 说明 |
|--------|----------|----------|------|
| 基础奖池 | SSR | 90抽 | 每90抽必出SSR |
| 高级奖池 | SSR | 80抽 | 每80抽必出SSR |
| 限定奖池 | SSR | 60抽 | 每60抽必出SSR |
| 终极奖池 | UR | 180抽 | 每180抽必出UR |

### 7.2 保底概率公式

```
SSR概率 = BaseProb + (CurrentPulls / PityThreshold) × (100% - BaseProb)
```

### 7.3 保底触发流程

```
1. 玩家发起抽卡请求
2. 检查保底计数是否达到阈值
3. 若达到阈值，强制抽取保底物品
4. 否则，按正常概率抽取
5. 抽取后重置/增加保底计数
6. 记录抽卡日志和保底进度
```

## 十、常用查询示例

### 10.1 查询玩家抽卡记录

```sql
SELECT ggl.*, gp.pool_name, gp.pool_type
FROM user_gacha_logs ggl
JOIN gacha_pools gp ON ggl.pool_id = gp.id
WHERE ggl.user_id = ?
ORDER BY ggl.created_at DESC
LIMIT 50;
```

### 10.2 查询玩家保底进度

```sql
SELECT ugp.*, gp.pool_name, gp.pool_type,
       ugp.pity_threshold - ugp.current_count as remain_count
FROM user_gacha_pity ugp
JOIN gacha_pools gp ON ugp.pool_id = gp.id
WHERE ugp.user_id = ? AND gp.is_active = 1;
```

### 10.3 抽卡概率计算

```sql
SELECT 
    ggi.quality,
    SUM(ggi.weight) as total_weight
FROM gacha_pool_items ggi
WHERE ggi.pool_id = ?
GROUP BY ggi.quality
ORDER BY ggi.quality;
```
