## 一、ER关系总览

                                 │              │  宝物实例表      │
                                 │              └────────┬────────┘

                        ┌────────┴────────┐              │
                        │   user_hero_artifacts          │
                        │   英雄宝物关联表                │

## 十七、数据库设计规范

### 17.1 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 表名 | 小写下划线命名，英文单词单数 | hero → heroes |
| 字段名 | 小写下划线命名 | hero_name, user_id |
| 主键 | id | id |
| 外键 | [表名]_id | hero_id, user_id |
| 索引名 | idx_[表名]_[字段] | idx_heroes_quality |
| 唯一索引 | uk_[表名]_[字段] | uk_user_heroes_user_hero |

### 17.2 数据类型规范

| 数据类型 | 使用场景 | 示例 |
|----------|----------|------|
| BIGINT | 主键、玩家ID、大数值 | id, user_id, diamond |
| INT | 普通数值、等级、计数 | level, quantity, gold |
| TINYINT | 枚举、状态标记 | quality, star_level, is_locked |
| DECIMAL(10,4) | 属性加成百分比 | hp_bonus_percent |
| DECIMAL(10,2) | 品质系数 | quality_coefficient |
| VARCHAR(50) | 代码、简短名称 | hero_code, item_code |
| VARCHAR(100) | 名称、描述 | hero_name, description |
| VARCHAR(500) | 路径、详细描述 | icon_url, skill_desc |
| TEXT | 长文本描述 | description, skill_desc |
| JSON | 结构化数据 | effect_params, before_stats |
| DATETIME | 时间戳 | created_at, updated_at |

### 17.3 索引设计原则

| 原则 | 说明 |
|------|------|
| 外键必索引 | 所有外键字段必须建立索引 |
| 查询条件字段 | WHERE/JOIN频繁使用的字段 |
| 排序字段 | ORDER BY频繁使用的字段 |
| 唯一约束 | 唯一性约束字段 |
| 组合索引 | 遵循最左前缀原则 |

### 17.4 性能优化建议

| 建议 | 说明 |
|------|------|
| 分库分表 | 玩家数据量大时按user_id哈希分库 |
| 读写分离 | 配置主从复制分离读写压力 |
| 缓存策略 | 热点数据(英雄基础表、道具基础表)使用Redis缓存 |
| 冷热分离 | 历史突破记录归档到冷库 |
| 定期清理 | 清理长期不活跃玩家的临时数据 |
