# 排行榜系统设计文档

## 一、系统概述

### 1.1 系统定位
排行榜系统是《苍穹劫·摸金传人》的核心竞争与社交展示系统，通过多维度排行展示激发玩家竞争意识，驱动付费与养成投入。系统设计兼顾公平竞技与娱乐展示，提供丰富的排名奖励激励玩家攀升榜单。

### 1.2 设计目标
- **竞争激励**：通过排行榜激发玩家竞争动力
- **目标驱动**：为玩家提供明确的成长目标
- **社交展示**：满足玩家炫耀与社交需求
- **奖励激励**：通过排名奖励引导投入
- **公平竞技**：确保排行榜的公正可信

## 三、详细功能设计

### 3.1 排行榜配置表结构

```yaml
ranking_config:
  榜单配置:
    ranking_id: "RANK_POWER"
    ranking_name: "战力排行榜"
    ranking_type: "power"          # power/progress/pvp/cultivate/resource/social
    ranking_subtype: "total"        # 子类型
    
  显示配置:
    show_count: 100                 # 显示前100名
    page_size: 20                   # 每页数量
    show_self_rank: true           # 显示自己排名
    show_nearby: true              # 显示附近排名
    
  刷新配置:
    refresh_type: "realtime"       # realtime/timed/seasonal
    refresh_interval: 3600          # 刷新间隔(秒)
    cache_expire: 300              # 缓存过期(秒)
    
  奖励配置:
    reward_type: "seasonal"        # instant/seasonal
    season_days: 30                # 赛季天数
    reward_distribution: "end"      # start/middle/end
    
  可见性配置:
    visibility: "public"           # public/guild/private
    min_level: 1                  # 查看最低等级
    min_display_level: 10          # 上榜最低等级
```

### 3.2 排行榜计算规则

#### 3.2.1 战力排行榜
```yaml
战力计算规则:
  
  总战力:
    = 主角战力 + 主力阵容战力 + 装备加成
    
  战力更新:
    - 实时监听战力变化
    - 变化超过阈值则更新
    - 离线玩家战力保持
    - 防作弊检测
    
  榜单限制:
    - 玩家等级≥10级
    - 上榜需消耗一次计算
    - 战力为0不参与排名
```

#### 3.2.2 关卡进度排行榜
```yaml
关卡计算规则:
  
  进度分计算:
    = 主线章节×100 + 关卡编号×10 + 通关星数
    
  进度更新:
    - 通关时立即更新
    - 优先显示最高进度
    - 同进度按通关时间排序
    
  榜单限制:
    - 玩家等级≥5级
    - 必须通关第1章才算入
```

### 3.3 排行榜防作弊机制

```yaml
防作弊规则:

  1. 战力异常检测
     - 战力增长速率限制
     - 战力数据交叉验证
     - 异常账号标记封禁
     
  2. 榜单刷新限制
     - 单玩家刷新频率限制
     - 同IP多账号限制
     - 数据一致性校验
     
  3. 作弊处罚
     - 榜单除名
     - 排行榜禁入
     - 账号封禁
     
  4. 申诉机制
     - 误判申诉入口
     - 数据复核流程
```

## 五、数据表设计

### 5.1 排行榜配置表 (rankings)

```sql
CREATE TABLE `rankings` (
  `ranking_id` VARCHAR(32) PRIMARY KEY COMMENT '排行榜ID',
  `ranking_name` VARCHAR(64) NOT NULL COMMENT '排行榜名称',
  `ranking_type` VARCHAR(16) NOT NULL COMMENT '排行榜类型',
  `ranking_subtype` VARCHAR(32) COMMENT '子类型',
  `description` VARCHAR(256) COMMENT '描述',
  `show_count` INT DEFAULT 100 COMMENT '显示数量',
  `page_size` INT DEFAULT 20 COMMENT '分页大小',
  `show_self_rank` TINYINT DEFAULT 1 COMMENT '显示自己排名',
  `show_nearby` TINYINT DEFAULT 1 COMMENT '显示附近排名',
  `nearby_range` INT DEFAULT 5 COMMENT '附近范围',
  `refresh_type` VARCHAR(16) DEFAULT 'realtime' COMMENT '刷新类型',
  `refresh_interval` INT DEFAULT 3600 COMMENT '刷新间隔(秒)',
  `cache_expire` INT DEFAULT 300 COMMENT '缓存过期(秒)',
  `reward_type` VARCHAR(16) DEFAULT 'seasonal' COMMENT '奖励类型',
  `season_days` INT DEFAULT 30 COMMENT '赛季天数',
  `reward_distribution` VARCHAR(16) DEFAULT 'end' COMMENT '奖励发放时机',
  `visibility` VARCHAR(16) DEFAULT 'public' COMMENT '可见性',
  `min_level` INT DEFAULT 1 COMMENT '查看最低等级',
  `min_display_level` INT DEFAULT 10 COMMENT '上榜最低等级',
  `sort_order` INT DEFAULT 0 COMMENT '排序',
  `is_active` TINYINT DEFAULT 1 COMMENT '是否启用',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX `idx_type` (`ranking_type`),
  INDEX `idx_active` (`is_active`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='排行榜配置表';
```

### 6.2 具体关联设计

#### 与战斗系统关联
- 战力数据实时同步到排行榜
- 阵容变化触发排名重算
- 战斗结算更新战力榜
- 历史最高战力记录追踪

#### 与养成系统关联
- 角色等级/品阶影响排名
- 装备数据同步到榜单展示
- 养成进度计入养成排行榜
- 养成成就触发榜单变化

#### 与竞技系统关联
- 竞技场排名同步到排行榜
- 巅峰赛成绩计入综合排名
- 赛季重置与竞技赛季同步
- 竞技奖励联动排行榜奖励

#### 与关卡系统关联
- 通关进度计入关卡排行榜
- 星级评价影响排名权重
- 隐藏关卡特殊排名
- 关卡通关时间记录

#### 与邮件系统关联
- 排名奖励通过邮件发放
- 排名变化通知邮件
- 赛季奖励结算邮件
- 榜单异常通知邮件

#### 与社交系统关联
- 公会成员战力展示
- 公会战力排行
- 好友战力对比
- 跨服战力排名

```yaml
  4. 预计算

     - 缓存预热机制
```

### 7.2 排行榜算法
```yaml
排名算法:
  1. 战力排名
     - 主战力 = sum(角色战力) + 装备加成
     - 同战力按时间排序
     
  2. 综合排名
     - 加权求和：战力×0.4 + 进度×0.3 + 竞技×0.3
     - 归一化处理
     
  3. 防刷机制
     - 异常数据过滤
     - 增长率限制
     - 交叉验证
```
