## 一、设计概述

### 1.1 编队系统定位

编队系统是连接英雄系统、战斗系统和关卡系统的核心枢纽，负责管理玩家上阵英雄的组合配置。玩家通过编队系统组建多套阵容，以应对不同关卡和玩法的挑战。

### 1.3 设计目标

| 目标 | 说明 |
|------|------|
| **便捷操作** | 支持拖拽、点击等多种编队操作方式 |
| **策略深度** | 支持羁绊系统，提供阵容组合策略 |
| **灵活配置** | 支持多套编队，满足不同关卡需求 |

## 二、编队基本规则

### 2.1 编队数量

| 配置项 | 设计值 | 说明 |
|--------|--------|------|
| 最大编队数量 | **5套** | 满足PVE/PVP多种场景需求 |
| 默认编队数量 | **3套** | 新玩家初始拥有3套 |
| 编队解锁条件 | 1-3套默认，4-5套需达到指定等级（30级/60级） |

**编队用途分配建议：**

| 编队序号 | 名称 | 用途 | 推荐场景 |
|----------|------|------|----------|
| 队伍1 | 主队 | 主线推图 | 普通/精英关卡 |
| 队伍2 | 副本队 | 挑战副本 | Boss关卡、高难度副本 |
| 队伍3 | 活动队 | 活动专用 | 限时活动关卡 |
| 队伍4 | 竞技队 | PVP竞技 | 竞技场、排行榜 |
| 队伍5 | 备用队 | 自由配置 | 自定义用途 |

### 2.2 队伍人数

| 配置项 | 设计值 | 说明 |
|--------|--------|------|
| 每队最大人数 | **5人** | 符合放置卡牌标准配置 |
| 最低上阵人数 | **1人** | 单英雄可通关（难度较高） |
| 推荐上阵人数 | **4-5人** | 最佳战斗配置 |

**站位设计：**

| 站位 | 位置说明 | 特性 |
|------|----------|------|
| 前排 | 位置1、2 | 优先受到攻击，承伤角色 |
| 后排 | 位置3、4、5 | 输出/辅助角色，安全输出 |

```
站位示意图：
┌─────────────────────────────────────┐
│           【敌方区域】              │
│    ┌───┐   ┌───┐   ┌───┐          │
│    │ E1│   │ E2│   │ E3│          │
│    └───┘   └───┘   └───┘          │
├─────────────────────────────────────┤
│           【我方区域】              │
│  ┌───┐   ┌───┐                      │
│  │ 1 │   │ 2 │    ← 前排(优先受击)   │
│  └───┘   └───┘                      │
│  ┌───┐   ┌───┐   ┌───┐             │
│  │ 3 │   │ 4 │   │ 5 │    ← 后排   │
│  └───┘   └───┘   └───┘             │
└─────────────────────────────────────┘
```

### 2.3 队伍命名

| 配置项 | 设计值 | 说明 |
|--------|--------|------|
| 队伍名称 | 玩家自定义（默认名称） | 6个汉字或12个字符 |
| 默认名称 | "队伍一" ~ "队伍五" | 方便辨识 |
| 名称修改 | 支持随时修改 | 无消耗 |

### 2.4 默认队伍

| 配置项 | 设计值 | 说明 |
|--------|--------|------|
| 默认队伍标记 | 1个 | 标记为默认队伍 |
| 默认队伍用途 | 进入关卡时的默认选择 | 可快速切换 |
| 默认切换 | 支持切换默认队伍 | 无消耗 |

## 四、编队操作

### 4.1 添加英雄

**方式一：点击添加**

```
步骤1：点击英雄列表中的英雄
        ↓
步骤2：点击阵容空槽位
        ↓
步骤3：英雄上阵成功
```

**方式二：拖拽添加**

```
步骤1：长按英雄卡片（200ms）
        ↓
步骤2：拖动到阵容空槽位
        ↓
步骤3：松开手指，英雄上阵成功
```

**限制规则：**

| 规则 | 说明 |
|------|------|
| 英雄不可重复上阵 | 同一英雄只能在一套阵容中 |
| 最低1人上阵 | 可保留空位 |
| 满员后不可添加 | 需先移除其他英雄 |

### 4.2 移除英雄

**方式一：点击移除**

```
步骤1：点击阵容中已上阵的英雄
        ↓
步骤2：弹出操作菜单
        ↓
步骤3：选择"移除"
```

**方式二：拖拽移除**

```
步骤1：长按已上阵英雄
        ↓
步骤2：拖动到英雄列表区域
        ↓
步骤3：松开手指，英雄移除
```

### 4.3 交换位置

**方式一：按钮交换**

```
步骤1：点击第一个英雄（选中状态）
        ↓
步骤2：点击第二个英雄
        ↓
步骤3：两个英雄位置交换
```

**方式二：拖拽交换**

```
步骤1：长按第一个英雄
        ↓
步骤2：拖动到第二个英雄位置
        ↓
步骤3：松开手指，位置交换
```

### 4.4 一键清空

```
点击"一键清空"按钮
        ↓
弹出确认弹窗："确定要清空当前阵容吗？"
        ↓
点击"确认" → 阵容清空
点击"取消" → 操作取消
```

### 4.5 一键推荐

**推荐算法：**

```
输入：玩家拥有的英雄列表
      关卡推荐战力（可选）
      羁绊需求（可选）
      ↓
处理：
1. 按战力排序，取前5名
2. 考虑传承均衡分布
3. 考虑羁绊组合加成
4. 考虑站位需求（前排坦克/后排输出）
      ↓
输出：推荐阵容
```

**推荐结果展示：**

### 6.2 战力显示

**队伍总战力计算公式：**

```
队伍总战力 = Σ(英雄战力) × 羁绊加成系数

羁绊加成系数 = 1 + Σ(各羁绊加成)
            = 1 + 地脉奇术(15%) + ... 
```

**单英雄战力计算：**

```
英雄战力 = 基础战力 × 等级系数 × 星级系数 × 突破系数 × 宝物加成
```

（具体公式参见《战力计算公式设计.md》）

### 6.3 传承分布

**传承类型定义：**

| 传承ID | 传承名称 | 特点 | 代表英雄 |

| 1 | 星纹摸金 | 精准探测、宝物加成 | 铁柱 |
| 2 | 璇玑搬山 | 力量型、前排坦克 | 搬山客 |
| 3 | 辰宿卸岭 | 敏捷型、暴击输出 | 摸金手 |
| 4 | 天枢发丘 | 智慧型、辅助控制 | 星祭司 |

### 8.2 羁绊效果计算

```
队伍最终属性 = 基础属性 × (1 + Σ羁绊加成)

示例：
基础属性 = 1000
羁绊加成 = 地脉奇术(+15%) + 传承加成(+10%) + 四人同心(+10%)
最终属性 = 1000 × (1 + 0.15 + 0.10 + 0.10) = 1000 × 1.35 = 1350
```

### 8.3 羁绊激活检测

```python
def check_team_bonds(team_heroes):
    """
    检测阵容羁绊激活情况
    输入：team_heroes - 阵容英雄列表
    输出：activated_bonds - 已激活羁绊列表
    """
    activated_bonds = []
    
    # 1. 检测固定羁绊
    for bond in fixed_bonds:
        if all(hero_id in team_heroes for hero_id in bond.required_heroes):
            activated_bonds.append(bond)
    
    # 2. 检测传承羁绊
    for heritage in heritages:
        count = sum(1 for hero in team_heroes if hero.heritage == heritage.id)
        if count >= 2:
            activated_bonds.append(create_heritage_bond(heritage, count))
    
    # 3. 检测阵营羁绊
    for faction in factions:
        count = sum(1 for hero in team_heroes if hero.faction == faction.id)
        if count >= faction.min_count:
            activated_bonds.append(create_faction_bond(faction, count))
    
    return activated_bonds
```

| 释放失败 | 英雄卡片弹回原位置 |

## 十三、错误处理

### 13.1 常见错误

| 错误类型 | 错误码 | 错误信息 | 处理方式 |
|----------|--------|----------|----------|
| 英雄不存在 | E001 | 英雄不存在或已删除 | 提示并移除该位置 |
| 英雄未拥有 | E002 | 该英雄尚未获得 | 提示并禁止添加 |
| 英雄已上阵 | E003 | 该英雄已在其他阵容中 | 提示并禁止添加 |
| 阵容已满 | E004 | 阵容已满员 | 提示先移除其他英雄 |
| 阵容为空 | E005 | 阵容至少需要1名英雄 | 禁止开始战斗 |
| 战力不足 | W001 | 战力低于推荐值，是否继续？ | 显示警告，可继续挑战 |

### 13.2 边界处理

| 场景 | 处理方式 |
|------|----------|
| 英雄被消耗/删除 | 自动从阵容移除该英雄 |
| 英雄被锁定 | 阵容显示锁定图标，不可编辑 |
| 新英雄解锁传承羁绊 | 立即激活并显示提示 |
| 编队数量已达上限 | 禁止创建新编队，提示解锁条件 |
