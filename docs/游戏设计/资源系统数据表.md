# 《苍穹劫·摸金传人》资源系统数据表设计

> 数据库设计文档 - 版本 1.0

## 一、设计概述

### 1.1 资源系统架构

```
资源系统
├── 货币系统（虚拟货币管理）
├── 体力系统（行动力管理）
├── 道具系统（物品管理）
└── 背包系统（存储管理）
```

### 1.2 表清单

| 序号 | 表名 | 说明 |
|------|------|------|
| 1 | currencies | 货币配置表 |
| 2 | user_currencies | 玩家货币持有表 |
| 3 | stamina_config | 体力恢复配置表 |
| 4 | user_stamina_logs | 体力消耗记录表 |
| 5 | user_daily_resources | 每日资源领取表 |

## 八、常用查询示例

### 8.1 查询玩家所有货币

```sql
SELECT c.currency_code, c.currency_name, c.icon_url, 
       uc.amount, uc.total_acquired, uc.total_consumed
FROM currencies c
LEFT JOIN user_currencies uc ON c.id = uc.currency_id 
    AND uc.user_id = ?
WHERE c.is_active = 1
ORDER BY c.sort_order;
```

### 8.2 查询玩家体力状态

```sql
SELECT 
    sc.stamina_type,
    COALESCE(us.current_value, sc.max_stamina) as current_stamina,
    sc.max_stamina,
    sc.recover_interval,
    (SELECT COUNT(*) FROM user_stamina_logs 
     WHERE user_id = ? AND stamina_type = sc.stamina_type 
     AND action_type = 'STAMINA_BUY' 
     AND DATE(created_at) = CURDATE()) as buy_count
FROM stamina_config sc
LEFT JOIN (
    SELECT u1.user_id, u1.stamina_type, 
           SUM(CASE WHEN action_type LIKE '%CONSUME%' THEN -stamina_change ELSE stamina_change END) as current_value
    FROM user_stamina_logs u1
    WHERE u1.user_id = ?
    GROUP BY u1.user_id, u1.stamina_type
) us ON sc.stamina_type = us.stamina_type
WHERE sc.is_active = 1;
```

### 8.3 查询每日资源状态

```sql
SELECT resource_type, current_value, max_value
FROM user_daily_resources
WHERE user_id = ? AND date_key = CURDATE();
```
