# 《苍穹劫》机关解谜系统设计

> 核心设计文档 - 版本 1.0
> 创建日期：2026年4月18日

---

## 一、机关类型总览

| 类型 | 核心机制 | 是否需要英雄 | 解法 |
|------|----------|-------------|------|
| **运气型** | 随机/概率 | 无关 | 选择选项，结果随机 |
| **知识型** | 知识/逻辑/数学 | 有影响但不决定 | 答题，英雄提供提示 |
| **英雄型** | 英雄能力 | 决定性因素 | 选对英雄 = 完美破解 |

---

## 二、运气型机关

### 2.1 设计原则

- 纯随机，与英雄无关
- 所有玩家体验一致
- 增加探索的不确定性和刺激感

### 2.2 机关示例

#### 三岔路口

```
【场景】
三条墓道向前延伸，黑暗中看不到尽头...

【AI图片】
三条幽暗的墓道入口

【选项】
A. 走左边
B. 走中间  
C. 走右边

【结果】（随机分配）
├── 正确道路 → 继续前进
├── 陷阱道路 → 掉血15%
└── 死胡同 → 浪费时间，回到原点
```

#### 神秘宝箱

```
【场景】
一个布满灰尘的宝箱，不知道有没有陷阱...

【AI图片】
古老青铜宝箱

【选项】
A. 直接打开（50%有陷阱）
B. 小心检查后再开
C. 放弃

【结果】
├── A有陷阱 → 掉血，获得奖励
├── A无陷阱 → 获得奖励
├── B → 安全，获得80%奖励
└── C → 无收益
```

#### 骰子石板

```
【场景】
一块石板上刻着数字，似乎需要掷骰子决定命运...

【选项】
A. 掷骰子（1-3：失败，4-6：成功）
B. 不掷，绕道走

【结果】
├── 成功 → 获得奖励
├── 失败 → 触发陷阱
└── 绕道 → 无收益无风险
```

## 三、知识型机关

### 3.1 设计原则

- 考验知识/逻辑/数学能力
- 英雄可以提供提示，但不直接给出答案
- 有知识就能答对，没有也能靠提示辅助

### 3.2 英雄影响

| 英雄类型 | 影响 | 效果 |
|----------|------|------|
| **高智力英雄** | 提供提示 | 显示解题思路/排除错误选项 |
| **传承匹配英雄** | 降低难度 | 题目难度下降一级 |
| **无相关英雄** | 无影响 | 靠自己答题 |

### 3.3 机关示例

#### 星盘计算（数学型）

```
【场景】
青铜星盘上刻着星辰位置，需要计算角度...

【英雄影响】
├── 有星纹摸金传承：提供公式提示
├── 智力>80的英雄：显示1个错误选项
└── 无相关英雄：无提示

【题目】
"北斗七星与北极星的夹角是30度，
猎户座在北斗七星东侧60度，
问：猎户座与北极星的夹角是多少？"

选项：
A. 30度
B. 60度
C. 90度
D. 120度

【有摸金传承提示】
"提示：三角形内角之和为180度"

【答案】C. 90度
```

#### 五行推演（逻辑型）

```
【场景】
石壁上的五行符文需要按正确顺序激活...

【英雄影响】
├── 有璇玑搬山传承：显示五行相生相克关系图
├── 智力>80：排除1个错误选项
└── 无相关英雄：无提示

【题目】
"铭文写着：'顺相生而开，逆相克而闭'
请选择正确的激活顺序："

选项：
A. 木→火→土→金→水（相生顺序）
B. 水→火→金→木→土（相克顺序）
C. 木→土→水→火→金（混合顺序）
D. 金→木→水→火→土（另一顺序）

【有搬山传承提示】
"相生：木→火→土→金→水→木
相克：木→土→水→火→金→木"

【答案】A. 相生顺序
```

#### 古文解读（知识型）

```
【场景】
一块石碑上刻着古代文字...

【英雄影响】
├── 有天枢发丘传承：提供翻译提示
├── 智力>80：显示关键词解释
└── 无相关英雄：无提示

【题目】
"石碑上写着：
'甲子之后，又三旬，天星动'
问：这是多少年后？"

选项：
A. 30年
B. 60年
C. 90年
D. 120年

【有发丘传承提示】
"甲子 = 60年，三旬 = 30年"

【答案】C. 90年
```

### 3.4 题目分类

| 类型 | 考察内容 | 示例 |
|------|----------|------|
| **数学计算** | 计算、几何、推理 | 角度计算、数字规律 |
| **逻辑推理** | 逻辑链条、条件判断 | 五行推演、顺序推理 |
| **科普知识** | 传统文化、历史 | 天干地支、古代常识 |
| **盗墓知识** | 盗墓文化、设定 | 传承规矩、风水知识 |

## 四、英雄型机关

### 4.1 设计原则

- 英雄能力决定结果
- 不需要答题，只需要选择
- 选择正确的传承英雄 = 完美破解

### 4.2 机关示例

#### 铁闸门

```
【场景】
一道锈迹斑斑的铁门挡住去路...

【选项】
A. 用力推开 → 关联：辰宿卸岭传承（力量型）
B. 寻找机关 → 关联：璇玑搬山传承（机关型）
C. 硬闯过去 → 无关联

【结果表】
| 选择 | 有对应传承 | 无对应传承 |
|------|-----------|-----------|
| A | 成功，轻微消耗 | 成功，掉血20% |
| B | 完美成功 | 失败，无其他路 |
| C | 成功，掉血15% | 成功，掉血25% |
```

#### 星盘锁

```
【场景】
门上镶嵌着星盘，需要调整星座位置...

【选项】
A. 根据星象推算 → 关联：星纹摸金传承（星象型）
B. 用罗盘定位 → 关联：星纹摸金传承（方位型）
C. 用力量破坏 → 关联：辰宿卸岭传承（力量型）

【结果表】
| 选择 | 有对应传承 | 无对应传承 |
|------|-----------|-----------|
| A | 完美破解，发现暗格 | 50%成功/50%陷阱 |
| B | 完美破解 | 失败 |
| C | 成功，触发警报 | 成功，掉血+警报 |
```

#### 悬魂梯

```
【场景】
螺旋向下的石阶，似乎走不到尽头...

【选项】
A. 标记路线 → 关联：璇玑搬山传承（机关型）
B. 用罗盘指引 → 关联：星纹摸金传承（方位型）
C. 直接走下去 → 无关联

【结果表】
| 选择 | 有对应传承 | 无对应传承 |
|------|-----------|-----------|
| A | 完美脱出，获奖励 | 浪费火把，随机脱出 |
| B | 成功脱出 | 继续绕圈 |
| C | 消耗时间 | 消耗大量时间 |
```

#### 五行封印

```
【场景】
五个元素符文需要按顺序激活...

【选项】
A. 按五行相生顺序 → 关联：璇玑搬山传承（阵法型）
B. 用发丘印破解 → 关联：天枢发丘传承（封印型）
C. 强行打破 → 关联：辰宿卸岭传承（力量型）

【结果表】
| 选择 | 有对应传承 | 无对应传承 |
|------|-----------|-----------|
| A | 完美破解 | 50%正确/50%陷阱 |
| B | 完美破解，额外奖励 | 无效 |
| C | 成功，触发战斗 | 成功，掉血+战斗 |
```

## 六、机关分布建议

### 6.1 单关卡分布

```
【普通地宫】（3-5个机关）
├── 运气型：1个
├── 知识型：1个
└── 英雄型：1-3个

【精英地宫】（5-8个机关）
├── 运气型：1-2个
├── 知识型：2个
└── 英雄型：2-4个

【Boss门前】（必解机关）
└── 英雄型：1个（必须破解才能进入Boss战）
```

### 6.2 机关占比

| 机关类型 | 占比 |
|----------|------|
| 运气型 | 20% |
| 知识型 | 30% |
| 英雄型 | 50% |

## 七、破解评级系统

| 评级 | 条件 | 奖励 |
|------|------|------|
| **完美** | 英雄型：有传承且选对选项 | 全部奖励 + 隐藏宝箱 |
| **优秀** | 知识型：一次答对 | 全部奖励 |
| **良好** | 知识型：第二次答对 / 英雄型：无传承但选对选项 | 80%奖励 |
| **勉强** | 使用提示 / 有代价通过 | 50%奖励 |
| **失败** | 答错 / 选错 | 触发陷阱/战斗 |

## 设计复盘：重复感问题（2026-04-18）

### 问题现象
Demo体验中发现，同类机关重复出现太频繁，比如连续遇到多个"神秘宝箱"、"铁闸门"，体验机械。

### 问题根源
1. 地图用概率随机生成，机关数量不可控
2. 每类机关只设计了3种具体类型，但会无限重复抽取
3. 纯随机布局缺乏设计感

### 优化方向
1. **限量机制**：每种具体机关全局只出现1次
2. **池化设计**：设计更多机关种类（如每类10种），每局随机抽取部分
3. **结构化地图**：不再纯随机铺满，设计区域/房间概念
4. **事件密度控制**：降低机关数量，让每次遭遇更有记忆点

### 待确认
- 地宫是单层还是多层结构？
- 每层预期探索时长？
- 机关是"必经之路"还是"可选支线"？

## 叙事包装问题（2026-04-18）

### 问题现象
迷宫纯机制，缺乏故事带入感，不知道"为什么在这里""在探索什么"。

### 解决方案框架

#### 1. 进入前叙事
- 任务引入：下地宫的目的（寻人/寻宝/调查/逃亡）
- 地宫背景：历史与传说（某朝古墓/废弃祭坛/地下城）
- 队伍动机：四人为何组队

#### 2. 探索中叙事
- **环境碎片**：随机触发的小剧情
  - 石碑文字：残留的历史记载
  - 壁画：画面叙事
  - 骸骨：暗示发生过什么
  - 破碎陶罐/腐烂木门：年代感
  
- **区域描述**：进入新区域时的氛围文字
- **发现描述**：物品有具体描写而非泛泛而谈

#### 3. 机关剧情化
- 每个机关有来历（守墓人设置/机关师遗产/陷阱）
- 破解结果有剧情反馈
- 与传承体系结合（摸金/搬山/卸岭/发丘各有发现能力）

#### 4. 地图结构叙事
- 区域划分：祭祀区/墓室区/宝藏区/机关区
- 环境暗示：地上的战斗痕迹、打翻的祭品、墙上的抓痕
- 探索拼图：通过碎片拼凑这里发生过什么

### 参考案例
- 《盗墓笔记》的环境描写与悬念铺设
- 《长生劫》的区域叙事与物品描述
- Souls系游戏的环境叙事（道具说明串联故事）

## Demo v6 待修复问题（2026-04-18）

### 初始视野缺失
- **问题**：刚进入地宫时玩家周围应该有两格初始视野，但目前开局只揭示了玩家所在格子，周围一片漆黑
- **原因**：`initGame` 中调用 `generateMap` 后没有调用 `revealArea`
- **解决方案**：在 `initGame` 的 `renderMap()` 之前调用 `revealArea(playerPos.x, playerPos.y, 2)`
- **状态**：✅ 已修复（2026-04-18）

## Demo v6 存档系统（2026-04-18）

### 实现方案
- **存储方式**：localStorage
- **版本控制**：`SAVE_VERSION = 1`，版本不匹配时自动清除旧存档

### 保存时机
1. 游戏开始时保存初始状态
2. 每次移动后自动保存
3. 页面关闭前保存（`beforeunload`）

### 清除时机
1. 关卡结算后清除
2. 存档版本不匹配时清除

### 存档内容
- 当前关卡ID
- 地图数据（尺寸、每个格子状态）
- 玩家状态（位置、HP、金币等）
- 进度数据（已破解机关、已发现故事碎片、目标完成情况）
- 保存时间戳

### UI交互
- 页面加载时检测存档，有存档则弹出提示框
- 显示存档信息：关卡名、进度、金币、保存时间
- 用户可选择"继续游戏"或"开始新游戏"

## 环境碎片的用途（待定，2026-04-18）

**已实现**：碎片收集界面，可回看已发现的碎片

**待设计的用途**：
1. **剧情拼图**：收集足够碎片，结算时解锁"真相"片段，揭示地宫背后的故事
2. **隐藏奖励**：收集满目标数量，结算时额外奖励金币
3. **线索提示**：某些碎片暗示隐藏区域/特殊机关的解法
4. **传承关联**：不同传承者可以"解读"碎片，获得额外信息
   - 摸金手看出风水布局
   - 星祭司解读符文含义
   - 搬山客发现机关痕迹
   - 铁柱感知危险气息

**后续决策点**：
- 碎片与结算奖励的关联方式
- 是否实现传承者解读功能
- 碎片对机关解谜的提示程度
