## 一、设计概述

### 1.1 奖励体系架构

```
奖励类型
├── 资源奖励（经验/金币/钻石）
├── 道具奖励（消耗品/材料）
├── 英雄奖励（整卡/碎片）
├── 宝物奖励（整件/碎片）
└── 特殊奖励（称号/头像框）
```

### 1.2 掉落机制说明

- **确定性掉落**：必定获得的基础奖励
- **概率性掉落**：根据概率表随机抽取
- **保底机制**：累计次数触发保底奖励

## 二、关卡掉落概率

### 2.1 掉落类型分类

| 掉落类型 | 说明 | 触发方式 |
|----------|------|----------|
| 必定掉落 | 每次通关必定获得 | 必然触发 |
| 概率掉落 | 根据概率随机决定 | 概率判定 |
| 额外掉落 | 通关评价触发 | S/A/B/C评价 |
| 首领掉落 | BOSS关卡专属 | 仅首领关 |

### 2.2 掉落概率公式

```
DropCheck(dropRate) = Random(0, 10000) < dropRate × 10000
```

- dropRate：掉落概率（0-1）
- 精度：小数点后4位

### 2.3 关卡掉落概率表

| 掉落物品 | 基础概率 | 章节加成 | VIP加成 |
|----------|----------|----------|---------|
| 英雄碎片 | 10% | +2%/章 | +5% |
| 宝物碎片 | 15% | +1%/章 | +3% |
| 经验药水 | 80% | - | - |
| 金币 | 100% | - | - |
| 突破石 | 20% | +1%/章 | +2% |

### 2.4 章节难度系数

| 章节 | 难度系数 | 掉落倍率 |
|------|----------|----------|
| 1-10章 | 1.0 | 1.0x |
| 11-20章 | 1.2 | 1.2x |
| 21-30章 | 1.5 | 1.5x |
| 31-40章 | 1.8 | 1.8x |
| 41-50章 | 2.2 | 2.2x |
| 51+章 | 2.5+ | 递增 |

### 2.5 掉落计算示例

**关卡：第15章-3**
- 基础掉落率：10%
- 章节加成：+10%（15-10=5章）
- VIP加成：+5%（VIP3）

```
实际掉落率 = 10% × 1.0 × 1.10 × 1.05 = 11.55%
```

## 三、金币掉落公式

### 3.1 金币基础产出

```
BaseGold(level) = Level × GoldPerLevel × StageMultiplier
```

| 参数 | 说明 | 默认值 |
|------|------|--------|
| GoldPerLevel | 每级金币基数 | 100 |
| StageMultiplier | 关卡系数 | 1.0-5.0 |

### 3.2 金币掉落表

| 关卡难度 | 金币基数 | 单次产出 |
|----------|----------|----------|
| 简单 | 500 | 500-1,000 |
| 普通 | 1,000 | 1,000-2,500 |
| 困难 | 2,000 | 2,000-5,000 |
| 噩梦 | 4,000 | 4,000-10,000 |
| 地狱 | 8,000 | 8,000-20,000 |

### 3.3 金币浮动计算

```
FinalGold = BaseGold × Random(MinFloat, MaxFloat)
```

- MinFloat = 0.9
- MaxFloat = 1.2

### 3.4 金币掉落示例

**第25章-5（普通难度）**
- 基础金币：2,000

```
FinalGold = 2000 × 1.1 = 2200（平均）
```

## 八、宝箱奖励公式

### 8.1 宝箱类型

| 宝箱类型 | 开启条件 | 奖励价值 |
|----------|----------|----------|
| 木箱 | 免费 | 1x |
| 银箱 | 30钻石 | 3x |
| 金箱 | 80钻石 | 8x |
| 钻石箱 | 200钻石 | 20x |
| 传说箱 | 500钻石 | 50x |

### 8.2 宝箱奖励计算

```
RewardValue = BaseReward × (1 + BonusRate) × LuckyBonus
```

### 8.3 幸运加成机制

连续开启宝箱可累积幸运值：

```
LuckyBonus = 1 + LuckyValue × LuckyMultiplier
LuckyValue = Min(LuckyValue + 0.05, LuckyCap)
```

| 宝箱类型 | 幸运累积 | 幸运上限 | 保底次数 |
|----------|----------|----------|----------|
| 金箱 | +5% | +50% | 20次 |
| 钻石箱 | +3% | +30% | 10次 |
| 传说箱 | +2% | +20% | 5次 |

### 8.4 保底机制

达到保底次数必定触发高价值奖励：

```
if (consecutive_no_rare >= pity_count) {
  grant_rare_reward();
  reset_consecutive_count();
}
```

## 十二、掉落系统使用指南

### 12.1 服务端掉落处理

```javascript
// 伪代码：关卡掉落处理
function processLevelDrops(userId, levelId) {
  const level = getLevelConfig(levelId);
  const drops = [];
  
  // 必定掉落
  drops.push(...processGuaranteedDrops(level.guaranteed_drops));
  
  // 概率掉落
  for (const pool of level.drop_pools) {
    if (checkDropRate(pool.drop_rate, userId.vip_level)) {
      const item = weightedRandomSelect(pool.items);
      drops.push(item);
    }
  }
  
  // 首领额外掉落
  if (level.is_boss) {
    drops.push(...processBossDrops(level));
  }
  
  return drops;
}
```

### 12.2 客户端展示

```javascript
// 伪代码：掉落展示
function formatDrops(drops) {
  return drops.map(drop => ({
    name: getItemName(drop.item_id),
    quantity: drop.quantity,
    rarity: getItemRarity(drop.item_id),
    icon: getItemIcon(drop.item_id)
  }));
}
```
