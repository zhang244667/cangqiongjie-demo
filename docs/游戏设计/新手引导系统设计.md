# 新手引导系统设计文档

## 一、系统概述

### 1.1 系统定位
新手引导系统是《苍穹劫·摸金传人》玩家进入游戏的第一体验，通过循序渐进的方式，帮助玩家快速理解游戏核心玩法，建立情感连接，降低流失率。系统采用分阶段、任务化的设计，让引导过程既有明确目标，又不失游戏乐趣。

### 1.2 设计目标
- **快速上手**：确保玩家10分钟内理解核心战斗
- **功能解锁**：通过引导逐步开放系统功能
- **情感建立**：通过剧情和角色建立游戏情感
- **目标引导**：为新玩家设立短期成长目标
- **流失预防**：通过奖励和成就感留住玩家

## 二、核心玩法设计

### 2.1 第一天引导流程（基础引导）

#### 2.1.1 角色创建阶段
```
引导步骤 1：角色创建
├── 展示角色选择界面
├── 选择职业：摸金校尉（输出）/ 搬山道人（坦克）/ 卸岭力士（辅助）
├── 输入角色名称
├── 引导点击「进入游戏」
└── 触发：进入新手村

├── 教学摇杆操作
├── 指定移动到发光点
├── 触发对话
└── 教学完成自动关闭
```

引导步骤 3：进入首个战斗
├── 引导点击「开始战斗」

├── 教学技能释放（能量满时高亮）

## 三、详细功能设计

### 3.1 引导配置表结构

```yaml
tutorial_config:
  基础配置:
    tutorial_id: "T001"
    tutorial_name: "战斗基础教学"
    tutorial_type: "force"        # force:强制 / skip:可跳过 / hide:隐藏
    tutorial_phase: 1             # 引导阶段 1-7
    tutorial_day: 1               # 对应天数
    
  触发条件:
    trigger_type: "quest_complete"  # 触发类型
    trigger_param: "QUEST_001"        # 触发参数
    trigger_delay: 0                   # 延迟触发(ms)
    
  步骤配置:
    steps:
      - step_id: 1
        step_type: "npc_dialog"       # 步骤类型
        step_content:                  # 步骤内容
          title: "欢迎来到苍穹"
          content: "这里是古墓入口，跟我来"
          speaker: "指引精灵"
        step_action:                   # 步骤动作
          type: "move_to"              # 移动
          target: "POINT_001"
        highlight_area: [x, y, w, h]   # 高亮区域
        next_auto: true                # 自动下一步
        auto_delay: 2000               # 自动延迟
        
  奖励配置:
    rewards:
      gold: 100
      exp: 50
      item_id: "ITEM_001"
      item_count: 1
      
  完成后跳转:
    next_tutorial: "T002"
```

### 3.2 引导类型定义

| 引导类型 | 说明 | 可跳过 | 奖励 |
|---------|------|-------|------|
| 强制引导 | 必须完成的教学 | 否 | 有 |
| 可跳过引导 | 可选择跳过 | 是 | 有 |
| 隐藏引导 | 暗中记录，不打断 | - | 无 |
| 提示引导 | 气泡提示，轻量 | 是 | 无 |
| 奖励引导 | 独立教学界面 | 是 | 有 |

### 3.3 引导触发条件

```yaml
触发类型枚举:
  
  1. 时间触发
     - login_time: 登录后X分钟触发
     - server_time: 特定时间点触发
  
  2. 行为触发
     - quest_complete: 完成任务触发
     - level_reach: 达到等级触发
     - system_unlock: 系统解锁触发
     - item_get: 获得物品触发
  
  3. 状态触发
     - status_change: 状态变化触发
     - progress_reach: 进度达成触发
  
  4. 事件触发
     - battle_end: 战斗结束触发
     - dialog_end: 对话结束触发
     - ui_open: 界面打开触发
```

### 3.4 引导流程控制

#### 3.4.1 步骤管理
```
引导步骤状态机：

  ┌─────────┐    触发条件满足    ┌─────────┐
  │  等待   │ ───────────────▶  │  进行中  │
  │  触发   │                   │          │
  └─────────┘                   └────┬─────┘
                                     │
                    ┌────────────────┼────────────────┐
                    │                │                │
              步骤完成          被跳过           强制中断
                    │                │                │
                    ▼                ▼                ▼
              ┌─────────┐      ┌─────────┐      ┌─────────┐
              │  下一   │      │  结束   │      │  暂停   │
              │  步骤   │      │  引导   │      │  等待   │
              └─────────┘      └─────────┘      └─────────┘
```

#### 3.4.2 打断与恢复机制
```
引导中断处理：
├── 强制打断：来电、退出游戏等 → 保存中断点，恢复时继续
├── 主动跳过：玩家主动跳过 → 记录跳过状态，次日可重置
├── 异常中断：网络断开等 → 自动保存，恢复后继续
└── 完成引导：正常完成 → 记录完成状态，解锁下一引导
```

#### 奖励配置规则
| 引导阶段 | 奖励类型 | 奖励价值 | 备注 |

## 五、数据表设计

### 5.1 引导配置表 (tutorial_config)

```sql
CREATE TABLE `tutorial_config` (
  `tutorial_id` VARCHAR(32) PRIMARY KEY COMMENT '引导ID',
  `tutorial_name` VARCHAR(64) NOT NULL COMMENT '引导名称',
  `tutorial_type` TINYINT NOT NULL COMMENT '引导类型：1强制/2可跳过/3隐藏/4提示',
  `tutorial_phase` TINYINT NOT NULL COMMENT '引导阶段1-7',
  `tutorial_day` TINYINT DEFAULT 1 COMMENT '对应天数',
  `trigger_type` VARCHAR(32) NOT NULL COMMENT '触发类型',
  `trigger_param` VARCHAR(256) COMMENT '触发参数',
  `trigger_delay` INT DEFAULT 0 COMMENT '触发延迟(ms)',
  `priority` INT DEFAULT 100 COMMENT '优先级',
  `parent_tutorial` VARCHAR(32) COMMENT '父引导ID',
  `is_parallel` TINYINT DEFAULT 0 COMMENT '是否并行引导',
  `parallel_group` VARCHAR(32) COMMENT '并行组ID',
  `unlock_condition` VARCHAR(512) COMMENT '解锁条件',
  `skip_available` TINYINT DEFAULT 0 COMMENT '是否可跳过',
  `skip_reward` VARCHAR(256) COMMENT '跳过奖励',
  `complete_reward` VARCHAR(512) COMMENT '完成奖励',
  `step_count` INT DEFAULT 1 COMMENT '步骤数量',
  `config_json` JSON COMMENT '完整配置JSON',
  `is_active` TINYINT DEFAULT 1 COMMENT '是否启用',
  `start_time` DATETIME COMMENT '开始时间',
  `end_time` DATETIME COMMENT '结束时间',
  `version` INT DEFAULT 1 COMMENT '版本号',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX `idx_phase` (`tutorial_phase`),
  INDEX `idx_day` (`tutorial_day`),
  INDEX `idx_trigger` (`trigger_type`),
  INDEX `idx_active` (`is_active`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='引导配置表';
```

### 6.2 具体关联设计

#### 与战斗系统关联
- 引导触发战斗教学事件
- 战斗系统提供引导用的教学关卡
- 战斗数据用于判断引导完成条件
- 战斗结算触发下一步引导

#### 与任务系统关联
- 引导流程中嵌入引导专属任务
- 引导任务完成后触发奖励发放
- 任务进度受引导状态影响
- 引导任务可配置为可跳过

#### 与邮件系统关联
- 离线奖励通过邮件发送
- 引导延迟奖励放入邮件附件
- 引导奖励领取记录同步
- 邮件提醒引导奖励可领取

#### 与社交系统关联
- 引导期好友数据隔离
- 公会系统引导在第四天解锁
- 社交引导与主引导并行
- 引导期社交限制提示

#### 与商城系统关联
- 首充引导触发商城界面
- 首充礼包配置与引导联动
- 商城消费计入引导目标
- 引导期商城UI简化展示

#### 与英雄系统关联
- 英雄教学是引导核心内容
- 引导赠送初始英雄
- 英雄升级/进阶教学嵌入引导
- 英雄收集是成就目标

### 6.3 事件接口设计

```yaml
引导事件接口:
  
  触发事件:
    - tutorial_start: 引导开始
    - tutorial_step_complete: 步骤完成
    - tutorial_complete: 引导完成
    - tutorial_skip: 引导跳过
    - tutorial_reward_claim: 奖励领取
  
  上报事件:
    - tutorial_enter: 进入引导
    - tutorial_exit: 退出引导
    - tutorial_duration: 引导时长
    - tutorial_skip_reason: 跳过原因
  
  回调事件:
    - on_step_complete: 步骤完成回调
    - on_guide_action: 引导动作回调
    - on_reward_claim: 奖励领取回调
```
