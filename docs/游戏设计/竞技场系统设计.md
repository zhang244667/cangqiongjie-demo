# 竞技场系统设计文档（1v1）

## 一、系统概述

### 1.1 系统定位
竞技场系统是《苍穹劫·摸金传人》的核心PVP玩法，采用1v1实时/异步对战模式，提供公平竞技与策略对抗的体验。系统通过段位机制、赛季制度和丰富奖励，驱动玩家持续参与PVP竞争。

### 1.2 设计目标
- **公平竞技**：确保对战双方实力接近
- **策略对抗**：深度技能组合与战斗策略
- **目标驱动**：段位提升提供长期追求
- **社交互动**：观战、挑战、好友对战
- **激励循环**：胜利奖励与赛季奖励

## 三、详细功能设计

### 3.1 竞技场配置表结构

```yaml
arena_config:
  基础配置:
    arena_id: "ARENA_1V1"
    arena_name: "苍穹竞技场"
    arena_type: "1v1"                   # 1v1/2v2/3v3
    
  战斗配置:
    max_battle_time: 300                # 最大战斗时长(秒)
    prepare_time: 15                    # 准备时间(秒)
    max_round_per_battle: 50            # 最大回合数
    
  匹配配置:
    match_timeout: 180                  # 匹配超时(秒)
    min_level: 15                       # 最低参与等级
    
  段位配置:
    initial_rank: "bronze"             # 初始段位
    star_protection: true              # 是否启用保星
    
  奖励配置:
    daily_free_challenge: 3            # 每日免费挑战次数
    daily_max_challenge: 6             # 每日最大挑战次数
    challenge_cost: 50                  # 额外挑战费用(钻石)
```

### 3.2 积分计算规则

```yaml
积分计算公式:

  基础积分:
    胜利: +20分
    失败: -10分
    
  段位加成:
    青铜: ×1.0
    白银: ×1.1
    黄金: ×1.2
    钻石: ×1.3
    大师: ×1.4
    宗师: ×1.5
    王者: ×1.6
    
  连胜加成:
    3连胜: +5额外积分
    5连胜: +10额外积分
    10连胜: +20额外积分
    
  积分上限:
    每日最大获得: 200分
    每日最大损失: 100分
```

### 3.3 防作弊机制

```yaml
防作弊规则:

  1. 战斗检测
     • 异常操作检测（秒放技能等）
     • 异常速度检测
     • 异常伤害检测
     
  2. 匹配检测
     • 同IP多账号检测
     • 异常匹配模式检测
     • 刷分行为检测
     
  3. 数据校验
     • 战斗结果服务端校验
     • 阵容数据加密传输
     • 战斗录像回放验证
     
  4. 处罚机制
     • 警告 → 禁赛 → 封号
     • 段位清除
     • 奖励回收
```

### 7.1 匹配算法伪代码
```yaml
匹配算法:
  function match_player(player):
    search_range = calculate_range(player, 0)
    wait_time = 0
    
    while wait_time < MAX_WAIT:
      candidates = find_candidates(player, search_range)
      if candidates:
        selected = select_best_match(candidates)
        if confirm_match(selected):
          return selected
      wait_time += SEARCH_INTERVAL
      search_range = calculate_range(player, wait_time)
    
    return null  # 匹配超时
```
