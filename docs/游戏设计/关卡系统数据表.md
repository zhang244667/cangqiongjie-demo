## 一、设计概述

### 1.1 关卡系统架构

```
关卡系统
├── 关卡配置（关卡定义）
├── 关卡掉落（奖励配置）
├── 关卡敌人（怪物配置）
├── 玩家进度（通关记录）
└── 关卡评价（星级评分）
```

### 1.2 表清单

| 序号 | 表名 | 说明 |
|------|------|------|
| 1 | levels | 关卡配置表 |
| 2 | level_drops | 关卡掉落表 |
| 3 | level_enemies | 关卡敌人配置表 |
| 4 | user_level_progress | 玩家关卡进度表 |
| 5 | user_level_stars | 玩家关卡星级表 |

## 九、常用查询示例

### 9.1 查询玩家章节进度

```sql
SELECT 
    l.chapter_id,
    l.chapter_name,
    COUNT(DISTINCT l.id) as total_levels,
    SUM(CASE WHEN ulp.status >= 1 THEN 1 ELSE 0 END) as cleared_levels,
    SUM(uls.star_1 + uls.star_2 + uls.star_3) as total_stars
FROM levels l
LEFT JOIN user_level_progress ulp ON l.id = ulp.level_id AND ulp.user_id = ?
LEFT JOIN user_level_stars uls ON l.id = uls.level_id AND uls.user_id = ?
WHERE l.level_type = 1 AND l.is_active = 1
GROUP BY l.chapter_id, l.chapter_name
ORDER BY l.chapter_id;
```

### 9.2 查询关卡详情（含掉落）

```sql
SELECT 
    l.*,
    ld.item_name,
    ld.drop_rate,
    ld.drop_count_min,
    ld.drop_count_max
FROM levels l
LEFT JOIN level_drops ld ON l.id = ld.level_id
WHERE l.id = ?;
```

### 9.3 查询可扫荡关卡

```sql
SELECT l.*
FROM levels l
JOIN user_level_progress ulp ON l.id = ulp.level_id AND ulp.user_id = ?
WHERE ulp.best_star >= 3 
    AND l.sweep_enabled = 1
    AND l.is_active = 1
ORDER BY l.chapter_id, l.level_index;
```
